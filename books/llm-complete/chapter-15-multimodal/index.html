多模态大模型

从文本到视觉到语音——理解和生成多种模态的统一架构

最后更新：2026年4月18日 多模态 Vision-Language 视觉编码器 语音 视频

从语言到多模态

人类的智能本质上是多模态的——我们同时通过视觉、听觉、触觉理解世界。纯文本 LLM 只利用了”语言”这一种信息源，而真实世界的信息远不止文本。

多模态大模型（Multimodal Large Language Models, MLLMs）的目标是构建能够理解和生成多种模态信息的统一模型：

输入模态:
├── 文本 (Text)
├── 图像 (Image)
├── 视频 (Video)
├── 音频/语音 (Audio/Speech)
└── 结构化数据 (Table/Code/...)

输出模态:
├── 文本 (Text)
├── 图像 (Image generation)
├── 语音 (Speech synthesis)
└── 行动 (Action, for agents)

架构范式

范式一：LLM + 外部编码器

这是目前最主流的架构——保持 LLM 不变，通过适配器接入其他模态的编码器。

图像 → [视觉编码器] → [投影层] ─┐
                                 ├── [LLM] → 文本输出
文本 → [Tokenizer] ─────────────┘

代表作：

模型视觉编码器LLM投影方式
LLaVACLIP ViT-LVicuna/LLaMA线性投影
LLaVA-1.5CLIP ViT-LVicunaMLP 投影
InternVLInternViT-6BInternLMQLLaMA
Qwen-VLViT-bigGQwen-7BCross-attention

范式二：Early Fusion（原生多模态）

将不同模态的 token 在输入层就混合，用同一个 Transformer 处理所有模态：

[图像 patch tokens] [文本 tokens] [音频 tokens]
        ↓                ↓              ↓
  统一的 Transformer（共享参数）
        ↓
    多模态输出

代表作：

模型方式特点
Fuyu图像 patch 直接线性投影无需单独的视觉编码器
Gemini原生多模态训练从预训练就混合多模态数据
GPT-4o统一模态文本/视觉/语音原生融合
Chameleon所有模态 tokenize离散 token 统一建模

Early Fusion vs. Late Fusion 对比：

维度Early FusionLate Fusion（编码器 + LLM）
跨模态理解更深入受限于投影层
训练成本极高（需多模态预训练）较低（冻结 LLM + 训练适配器）
灵活性需重新预训练可热插拔编码器
数据需求海量多模态数据百万级图文对即可

视觉编码器

CLIP（Contrastive Language-Image Pre-training）

CLIP（Radford et al., 2021, OpenAI）是多模态模型的基石——通过对比学习将图像和文本映射到共享的向量空间：

LCLIP=−1N∑i=1N[log⁡exp⁡(sim(Ii,Ti)/τ)∑j=1Nexp⁡(sim(Ii,Tj)/τ)]\mathcal{L}_{\text{CLIP}} = -\frac{1}{N} \sum_{i=1}^{N} \left[ \log \frac{\exp(\text{sim}(I_i, T_i) / \tau)}{\sum_{j=1}^{N} \exp(\text{sim}(I_i, T_j) / \tau)} \right]LCLIP​=−N1​∑i=1N​[log∑j=1N​exp(sim(Ii​,Tj​)/τ)exp(sim(Ii​,Ti​)/τ)​]

CLIP ViT-L/14@336px 是最常用的视觉编码器，输出 576 个 visual tokens（每个 1024 维）。

SigLIP

SigLIP（Zhai et al., 2023, Google）改进了 CLIP 的训练效率：

  • 用 sigmoid loss 替代 softmax loss，不需要全局 batch 内的负样本
  • 允许更大的 batch size 和更好的扩展性
  • 被 PaLI、Gemma 等 Google 系模型采用

视觉 Token 压缩

576 个 visual tokens 对 LLM 来说太多了（特别是多图/高分辨率场景）。压缩方法：

方法思路压缩比
Spatial pooling2x2 平均池化4x
Perceiver Resampler用可学习 query 做交叉注意力可变
C-Abstractor卷积 + 自适应池化可变
Dynamic resolution根据图像复杂度动态调整自适应

视觉语言训练流程

阶段一：预训练对齐（Alignment Pre-training）

用大量图文对训练投影层，让视觉编码器的输出与 LLM 的输入空间对齐：

数据: 图文对 (LAION-5B, CC3M, CC12M, etc.)
冻结: 视觉编码器 + LLM
训练: 只训练投影层 (MLP)
目标: 下一个 token 预测 (图像作为前缀)

阶段二：指令微调（Visual Instruction Tuning）

用高质量的视觉问答/对话数据微调整个模型：

数据: 视觉指令数据 (LLaVA-Instruct-150K, ShareGPT4V, etc.)
冻结: 视觉编码器（或也微调）
训练: 投影层 + LLM
目标: 视觉指令跟随

关键数据集

数据集规模类型用途
LAION-5B5.85B 图文对网页爬取预训练
ShareGPT4V1.2MGPT-4V 标注高质量指令微调
LLaVA-Instruct150K+GPT-4 生成视觉对话
TextVQA45K图中文字理解OCR 评估
DocVQA50K文档理解文档问答
MMMU11.5K多学科大学题综合评估

高分辨率处理

标准 CLIP ViT 只能处理 224×224 或 336×336 的图像，远不够实际使用。高分辨率方法：

动态分辨率切片

将高分辨率图像切成多个 patch，每个 patch 独立编码，再拼接：

原图 (1344×896)
  ↓ 切片
[patch_1] [patch_2] [patch_3]
[patch_4] [patch_5] [patch_6]
  ↓ 每个 patch 独立编码
[tokens_1] [tokens_2] ... [tokens_6] + [全局缩略图 tokens]
  ↓ 拼接
总共 576×7 = 4032 visual tokens

LLaVA-NeXT 和 InternVL 1.5 都使用类似方法，支持任意分辨率和纵横比。

视频理解

视频理解面临两个核心挑战：时序建模和token 爆炸。

帧采样策略

均匀采样: 从 N 帧中等间隔抽取 K 帧（最简单，常用）
关键帧提取: 基于场景变化检测选取关键帧
时间池化: 在时间维度上池化视觉 tokens

代表模型

模型方法最大帧数
Video-LLaVA均匀采样 + 投影8 帧
LLaVA-VideoSlowFast 采样64+ 帧
Qwen2-VL动态分辨率 + 时间编码视频长度
Gemini 1.5 Pro原生长上下文1小时+ 视频

语音多模态

语音理解

将语音编码器接入 LLM，实现语音指令理解：

语音 → [Whisper 编码器] → [适配器] → LLM → 文本回复

Whisper（Radford et al., 2022, OpenAI）是最常用的语音编码器，支持多语言语音识别和翻译。

语音生成

端到端的语音生成（不经过 TTS）：

模型方法特点
GPT-4o原生语音 token实时对话，保留情感和语调
VALL-E语音作为 token 序列3 秒 prompt 克隆声音
GLM-4-Voice端到端语音对话中英文

全模态 (Omni) 模型

真正的”全模态”模型能够在一个统一框架中处理所有模态的输入和输出：

Any-to-Any 模型:
├── 输入: 文本 + 图像 + 语音 + 视频
├── 处理: 统一 Transformer
└── 输出: 文本 + 图像 + 语音

GPT-4o 是这一方向的里程碑——同一个模型原生支持文本/视觉/语音的输入和输出，延迟低到可以实时对话。

Vision Arena

类似文本的 Chatbot Arena，Vision Arena 让用户用图像 + 文本提问，盲评两个模型的回复。

前沿方向

1. 统一生成模型

不仅理解多模态，还能生成多种模态：

  • 图像生成：Emu、DALL-E 3、Janus、Janus-Pro（DeepSeek，7B 开源，解耦编码器——理解用 SigLIP-L、生成用独立 VQ tokenizer，在 GenEval 上超越 DALL·E 3）
  • 视频生成：Sora、Kling、Veo
  • 3D 生成：从图像/文本生成 3D 模型

2. World Models

将多模态理解与物理世界建模结合：

  • 预测视频中物体的运动轨迹
  • 理解因果关系（推倒积木→积木散落）
  • 为机器人提供世界理解

3. 实时多模态交互

GPT-4o 开创的方向——毫秒级延迟的多模态实时对话：

用户: [语音] "帮我看看这道数学题"
      [图片] 📷 手写数学题
模型: [语音] "这是一道二次方程，让我帮你解..."
      [图片] 📝 解题步骤

2025-2026 前沿：原生多模态与统一生成

多模态 LLM 正在经历一次根本性的架构转变：从”拼接式”走向”原生式”。

原生多模态模型

传统方案（LLaVA 范式）是将预训练的视觉编码器与 LLM 拼接。2025-2026 年的新趋势是原生多模态——一个模型从训练开始就同时处理多种模态：

模型特点意义
GPT-4o原生支持文本/图像/语音的理解和生成首个大规模原生多模态商用模型
Gemini 3 Pro原生图像生成（类似 DALL-E 但集成在 LLM 中）证明 LLM 可以直接输出像素
Emu3纯 next-token prediction 统一文本/图像/视频Nature 2026 正式发表——首次在顶级期刊证明纯 next-token 范式可统一多模态建模
GLM-4.6V开源 106B 多模态 + 原生工具调用开源多模态的新标杆
Janus-Pro解耦视觉编码：理解和生成各用独立编码器证明”统一模型≠统一编码器”——解耦才是正确路径

统一理解与生成

最激动人心的方向是理解与生成的统一：同一个模型既能看懂图片，也能生成图片；既能听懂语音，也能合成语音。2025 年的综述研究（84 页，700+ 文献）将这一方向清晰地梳理为两大技术路线：

路线思路代表优势局限
纯 AR 统一所有模态 tokenize，统一 next-token predictionEmu3、Janus-Pro、Chameleon架构极简，训练流程统一生成质量受限于 VQ 重建精度
扩散+AR 混合理解用 AR，生成用 Diffusion headTransfusion、Show-o生成质量高（保留扩散模型优势）架构复杂，训练需协调两种目标

两条路线哪条会胜出尚无定论，但共识已清晰：Tokenizer 的概念正在从文本扩展到所有模态——视觉 token、音频 token、动作 token 统一进入同一个序列建模框架。Janus-Pro 的解耦编码器设计提供了一个折中思路：统一模型，但为理解和生成各自选择最优的编码方式。

核心洞察：多模态的终局可能不是”多个编码器的拼装”，而是一个真正的全模态 Transformer——文本、图像、音频、视频、3D 都是 token 序列，由同一个模型统一建模和生成。

章节小结

  1. 多模态是 LLM 的必经之路——纯文本模型只利用了人类信息的冰山一角
  2. LLM + 外部编码器是目前最实用的范式，Early Fusion 是长期方向
  3. CLIP/SigLIP 是视觉编码的基石，高分辨率处理是关键挑战
  4. 语音多模态正在从”文本中转”走向端到端原生处理
  5. 全模态 (Omni) 模型是终极目标——一个模型理解和生成所有模态

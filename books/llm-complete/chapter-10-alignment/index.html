第 10 章 第三部分：后训练与对齐

RLHF 与偏好对齐

从 RLHF 到 DPO 到 GRPO——让模型学会'什么该说什么不该说'

最后更新：2026年4月17日 RLHF DPO GRPO 对齐 奖励模型 PPO

为什么 SFT 不够

SFT 教会模型”怎么回答问题”，但没有教会它区分好回答和坏回答。一个 SFT 模型可能会：

  • 自信地输出错误信息（幻觉）
  • 回答本应拒绝的有害请求
  • 在多种合理回答中选择不够好的那个
  • 给出正确但不友好的回复

偏好对齐（Preference Alignment） 的目标是：让模型的输出不仅”正确”，而且”好”——符合人类的价值观和偏好。

RLHF：经典框架

三阶段流程

InstructGPT（Ouyang et al., 2022）奠定了 RLHF 的经典三阶段框架：

阶段 1: SFT
  预训练模型 → SFT 模型（指令跟随）

阶段 2: 奖励模型训练 (RM)
  人类标注偏好对 → 训练奖励模型

阶段 3: RL 优化
  SFT 模型 + 奖励模型 → PPO 优化 → 对齐后的模型

阶段 2：奖励模型（Reward Model）

奖励模型的训练数据是偏好对（preference pair）：

Prompt: "解释量子纠缠"
Response A: "量子纠缠是两个粒子之间的量子关联，当测量其中一个..."  ← 人类选择更好
Response B: "量子纠缠很复杂，你可以上网搜索一下。"               ← 人类认为更差

给定 prompt xxx 和两个回复 ywy_wyw​（preferred，更好的）和 yly_lyl​（rejected，更差的），奖励模型的训练目标基于 Bradley-Terry 模型：

LRM=−E(x,yw,yl)[log⁡σ(rϕ(x,yw)−rϕ(x,yl))]\mathcal{L}_{\text{RM}} = -\mathbb{E}_{(x, y_w, y_l)} \left[ \log \sigma\left( r_\phi(x, y_w) - r_\phi(x, y_l) \right) \right]LRM​=−E(x,yw​,yl​)​[logσ(rϕ​(x,yw​)−rϕ​(x,yl​))]

其中 rϕ(x,y)r_\phi(x, y)rϕ​(x,y) 是奖励模型对 prompt xxx 和回复 yyy 的打分，σ\sigmaσ 是 sigmoid 函数。

直觉上，这个 loss 鼓励奖励模型给好回复打更高的分，给坏回复打更低的分。

奖励模型的架构通常与策略模型相同（去掉 LM head，加一个线性层输出标量分数）：

输入序列 → Transformer → [最后一个 token 的 hidden state] → Linear → scalar reward

阶段 3：PPO 优化

PPO（Proximal Policy Optimization）（Schulman et al., 2017）是 RLHF 中最常用的 RL 算法。优化目标：

max⁡θEx∼D,y∼πθ(⋅∣x)[rϕ(x,y)−β⋅DKL(πθ∥πref)]\max_\theta \mathbb{E}_{x \sim \mathcal{D}, y \sim \pi_\theta(\cdot|x)} \left[ r_\phi(x, y) - \beta \cdot D_{\text{KL}}(\pi_\theta \| \pi_{\text{ref}}) \right]maxθ​Ex∼D,y∼πθ​(⋅∣x)​[rϕ​(x,y)−β⋅DKL​(πθ​∥πref​)]

两项的含义：

  • rϕ(x,y)r_\phi(x, y)rϕ​(x,y)：奖励模型的评分，越高越好
  • β⋅DKL\beta \cdot D_{\text{KL}}β⋅DKL​：KL 散度惩罚，防止策略模型偏离 SFT 模型太远

为什么需要 KL 惩罚？ 没有 KL 惩罚，模型会学会”奖励 hack”——找到奖励模型的漏洞，生成得分很高但实际质量很低的回复。比如不断重复某些高分词汇，或生成看似深奥实际无意义的文本。

PPO 的训练涉及四个模型同时在 GPU 上：

模型作用是否更新
策略模型 (Actor)生成回复是
价值模型 (Critic)估计 state value是
奖励模型 (RM)打分否（冻结）
参考模型 (Ref)计算 KL 惩罚否（冻结）

这意味着训练 PPO 的显存需求大约是 SFT 的 4 倍，这是 RLHF 落地困难的主要原因之一。

DPO：去掉奖励模型

DPO（Direct Preference Optimization）（Rafailov et al., 2023）是对 RLHF 的重大简化。核心思想是：可以跳过奖励模型训练和 RL 过程，直接用偏好数据优化策略模型。

数学推导

DPO 的关键洞察：RLHF 的最优策略有闭式解：

π∗(y∣x)=1Z(x)πref(y∣x)exp⁡(r(x,y)β)\pi^*(y|x) = \frac{1}{Z(x)} \pi_{\text{ref}}(y|x) \exp\left(\frac{r(x, y)}{\beta}\right)π∗(y∣x)=Z(x)1​πref​(y∣x)exp(βr(x,y)​)

将这个关系反转，可以用策略的概率来表达隐式的奖励：

r(x,y)=βlog⁡πθ(y∣x)πref(y∣x)+βlog⁡Z(x)r(x, y) = \beta \log \frac{\pi_\theta(y|x)}{\pi_{\text{ref}}(y|x)} + \beta \log Z(x)r(x,y)=βlogπref​(y∣x)πθ​(y∣x)​+βlogZ(x)

代入 Bradley-Terry 偏好模型，配分函数 Z(x)Z(x)Z(x) 消掉，得到 DPO loss：

LDPO=−E(x,yw,yl)[log⁡σ(βlog⁡πθ(yw∣x)πref(yw∣x)−βlog⁡πθ(yl∣x)πref(yl∣x))]\mathcal{L}_{\text{DPO}} = -\mathbb{E}_{(x, y_w, y_l)} \left[ \log \sigma\left( \beta \log \frac{\pi_\theta(y_w|x)}{\pi_{\text{ref}}(y_w|x)} - \beta \log \frac{\pi_\theta(y_l|x)}{\pi_{\text{ref}}(y_l|x)} \right) \right]LDPO​=−E(x,yw​,yl​)​[logσ(βlogπref​(yw​∣x)πθ​(yw​∣x)​−βlogπref​(yl​∣x)πθ​(yl​∣x)​)]

DPO 的局限

  1. 离线数据瓶颈：DPO 只能从已有的偏好对中学习，不能像 PPO 那样通过在线采样探索新的回复空间
  2. 过拟合风险：在小数据集上容易过拟合到特定的偏好模式
  3. 似然位移（likelihood displacement）：优化 ywy_wyw​ 和 yly_lyl​ 的概率比时，可能导致两者的绝对概率同时下降

DPO 变体家族

DPO 的简洁性催生了一大波变体：

IPO（Identity Preference Optimization）

IPO（Azar et al., 2023）指出 DPO 在偏好确定性很高时会过拟合，提出了正则化版本：

LIPO=E[(log⁡πθ(yw∣x)πref(yw∣x)−log⁡πθ(yl∣x)πref(yl∣x)−12β)2]\mathcal{L}_{\text{IPO}} = \mathbb{E} \left[ \left( \log \frac{\pi_\theta(y_w|x)}{\pi_{\text{ref}}(y_w|x)} - \log \frac{\pi_\theta(y_l|x)}{\pi_{\text{ref}}(y_l|x)} - \frac{1}{2\beta} \right)^2 \right]LIPO​=E[(logπref​(yw​∣x)πθ​(yw​∣x)​−logπref​(yl​∣x)πθ​(yl​∣x)​−2β1​)2]

KTO（Kahneman-Tversky Optimization）

KTO（Ethayarajh et al., 2024）灵感来自前景理论——人类对损失比收益更敏感。它不需要成对的偏好数据，只需要单独标注每个回复是好是坏：

LKTO=Eyw[1−σ(β⋅rw)]+λ⋅Eyl[1−σ(−β⋅rl)]\mathcal{L}_{\text{KTO}} = \mathbb{E}_{y_w} \left[ 1 - \sigma(\beta \cdot r_w) \right] + \lambda \cdot \mathbb{E}_{y_l} \left[ 1 - \sigma(-\beta \cdot r_l) \right]LKTO​=Eyw​​[1−σ(β⋅rw​)]+λ⋅Eyl​​[1−σ(−β⋅rl​)]

其中 λ>1\lambda > 1λ>1 反映了”损失厌恶”。

SimPO（Simple Preference Optimization）

SimPO（Meng et al., 2024）更进一步——连参考模型都不需要了。用平均 log 概率作为隐式奖励：

rSimPO(x,y)=1∣y∣log⁡πθ(y∣x)r_{\text{SimPO}}(x, y) = \frac{1}{|y|} \log \pi_\theta(y|x)rSimPO​(x,y)=∣y∣1​logπθ​(y∣x)

这消除了参考模型的显存开销，训练速度更快。

ORPO（Odds Ratio Preference Optimization）

ORPO（Hong et al., 2024）将 SFT 和偏好对齐合并为一个训练阶段：

LORPO=LSFT+λ⋅LOR\mathcal{L}_{\text{ORPO}} = \mathcal{L}_{\text{SFT}} + \lambda \cdot \mathcal{L}_{\text{OR}}LORPO​=LSFT​+λ⋅LOR​

GRPO：推理模型的对齐利器

GRPO（Group Relative Policy Optimization）（Shao et al., 2024，DeepSeek 团队）是专门为推理任务设计的对齐方法，在 DeepSeek-R1 中取得了显著效果。

核心思想

GRPO 不需要奖励模型，而是通过组内相对排序来计算奖励：

  1. 对每个 prompt xxx，采样一组回复 {y1,y2,...,yG}\{y_1, y_2, ..., y_G\}{y1​,y2​,...,yG​}
  2. 用规则 verifier（如数学题正确性检查）为每个回复打分 {r1,...,rG}\{r_1, ..., r_G\}{r1​,...,rG​}
  3. 将奖励标准化：r^i=ri−mean(r)std(r)\hat{r}_i = \frac{r_i - \text{mean}(r)}{\text{std}(r)}r^i​=std(r)ri​−mean(r)​
  4. 用标准化后的奖励做策略梯度更新

LGRPO=−1G∑i=1Gr^i⋅∑t=1∣yi∣log⁡πθ(yi,t∣x,yi,<t)\mathcal{L}_{\text{GRPO}} = -\frac{1}{G} \sum_{i=1}^{G} \hat{r}_i \cdot \sum_{t=1}^{|y_i|} \log \pi_\theta(y_{i,t} | x, y_{i,<t})LGRPO​=−G1​∑i=1G​r^i​⋅∑t=1∣yi​∣​logπθ​(yi,t​∣x,yi,<t​)

GRPO 特别适合数学和代码等有确定性验证器的任务——答案对就对，错就错，不需要模糊的人类偏好。

DAPO：工业级 GRPO

GRPO 的核心思想简洁优雅，但直接部署到大规模训练中会遇到几个工程问题。DAPO（Decoupled Clip and Dynamic Sampling Policy Optimization）（清华 AIR + 字节跳动 SIA-Lab，2025）针对这些问题提出了四项关键改进：

1. Clip 解耦。 标准 PPO/GRPO 对正负方向使用相同的 clip 比例。但正向梯度（鼓励好回复）被过度截断会减慢学习。DAPO 将上下 clip 比例分开——正方向 clip 放宽，允许更大的策略更新幅度。

2. Dynamic Sampling。 如果一组采样中所有回复全对（accuracy=1）或全错（accuracy=0），这组数据没有梯度信号——全对意味着没有改进空间，全错意味着模型在当前能力范围外。DAPO 在训练时过滤掉这两类 prompt，只在有难度梯度的样本上训练。

3. Token-level Loss。 GRPO 默认使用序列级 loss——长序列和短序列的 loss 权重相同。这导致长回复被过度惩罚（因为长序列的 token 级概率乘积天然更小）。DAPO 改为 token 级 loss，每个 token 的梯度贡献独立计算。

4. Overlong Penalty。 训练中超长回复（超出最大长度被截断）的处理：直接丢弃会浪费算力，不处理则鼓励冗长。DAPO 对截断的回复施加软惩罚——标记为负奖励但仍参与训练。

实际效果：在 AIME 2024 上，GRPO 约 30%+ → DAPO 达到约 50%。DAPO 已被字节、Qwen 等多个团队在生产中使用，开源实现集成在 veRL 框架中。

GSPO：从 Token 级到序列级裁剪

DAPO 的第三项改进（token-level loss）解决了长序列惩罚问题，但 GSPO（Group Sequence Policy Optimization）（阿里 Qwen 团队，2025）从另一个角度审视了这个问题。

GSPO 的核心洞察：PPO/GRPO/DAPO 在 token 级做裁剪时，一个序列中可能只有部分 token 被 clip，其余 token 仍参与梯度计算。这导致”半截”序列——模型被部分推向某个方向，但这个方向信号是不完整的。

GSPO 的解决方案直截了当：在序列级做裁剪——整个序列要么全部参与训练，要么完全不参与。用序列级重要性采样比率（而非 token 级）决定是否 clip。

这种设计让训练更稳定：每个参与训练的序列都提供完整的梯度信号。在 MATH 和 AIME 上，GSPO 的表现超过 DAPO，并已应用于阿里的 Qwen3 系列模型。

Constitutional AI（RLAIF）

Constitutional AI（Bai et al., 2022，Anthropic）用 AI 自身来替代人类标注偏好：

  1. 定义一组”宪法原则”（如”回复应该无害”、“应该诚实”）
  2. 让模型根据宪法原则对回复做自我批判和修改
  3. 让另一个模型根据宪法原则在两个回复中选择更好的
  4. 用 AI 偏好数据训练奖励模型（RLAIF）

这种方法的核心优势是可扩展性——不依赖人类标注员的速度限制。

对齐的挑战与前沿

Reward Hacking（奖励 Hack）

模型可能找到奖励模型的漏洞，生成高分但低质的回复。常见模式：

  • 冗长偏见：更长的回复往往得分更高，模型学会”凑字数”
  • 格式偏见：使用 Markdown 列表和加粗会提高分数
  • 逢迎偏见：同意用户的说法（即使是错的）得分更高

对策：

  • 多奖励模型集成
  • 奖励模型定期刷新
  • 长度归一化

Alignment Tax（对齐税）

对齐过程可能损害模型的基础能力。典型表现：

  • 过度拒绝无害请求（“我无法帮助…”）
  • 知识类任务性能下降
  • 创造力和多样性降低

迭代对齐

一次对齐往往不够。工业界通常采用迭代流程：

SFT → DPO/RLHF → 收集新偏好数据 → 再次 DPO/RLHF → ...

每一轮使用当前模型采样新的回复，收集新的偏好标注，持续改进。LLaMA 3 的技术报告中详细描述了他们的多轮迭代流程。

2025-2026 前沿：推理模型与可验证奖励 RL

后训练五代演进

回顾过去四年，后训练技术走过了清晰的五代演进——每一代都做了一次”减法”，去掉一个外部依赖：

世代代表方法年份关键”减法”需要的外部组件
第一代PPO (RLHF)2022—RM + Critic + Ref + 人工标注
第二代DPO2023去掉 Critic 和 RMRef + 离线偏好对
第三代GRPO2024去掉参考模型依赖（KL 松弛）规则 verifier
第四代DAPO / GSPO2025去掉静态采样假设规则 verifier
第五代RLVR / 自博弈2026去掉人工标注纯可验证奖励

这条演进线揭示了一个趋势：后训练正在从”人类密集监督”走向”自动化验证驱动”。当一个任务的正确性可以被程序验证时（数学、代码、形式逻辑），整条后训练流水线可以完全无人介入地运转。

对齐技术在 2025-2026 年经历了一次范式跃迁——基于可验证奖励的强化学习（RL with Verifiable Rewards） 成为训练推理模型的核心方法。

DeepSeek R1 的突破

DeepSeek R1 证明了一个惊人的发现：纯 RL 训练（无 SFT）就能让模型学会深度推理。在数学和代码等有客观答案的领域，模型通过 GRPO 自发学会了反思、回溯和自我纠错等推理行为。

关键技术要素：

  • 可验证奖励：数学题有标准答案、代码有测试用例——奖励信号无需人工标注
  • GRPO 变体：组内采样多个回复，用正确率作为奖励信号，无需训练独立的奖励模型
  • 推理 token 的自然涌现：模型自动学会了”让我再想想”、“这个方向不对，换一个思路”等推理策略

值得关注的是 ARPO（Agentic Reinforced Policy Optimization）（ICLR 2026）——它将强化学习扩展到多轮 Agent 交互场景。ARPO 利用熵驱动的自适应 rollout，在多轮工具调用中进行细粒度探索，仅用一半的工具调用预算即可达到更优性能。这标志着 RL 后训练从单轮问答走向复杂的序贯决策。

从对齐到能力提升

传统对齐（RLHF/DPO）的目标是不改变能力、只改变行为。但推理 RL 模糊了这条边界：

新的训练流水线

2026 年的典型后训练流程已经从三阶段扩展为四阶段：

SFT（格式对齐）
  → 推理 RL（能力提升，可验证奖励）
  → 偏好对齐（DPO/RLHF，人类偏好）
  → 安全对齐（红队测试 + 约束微调）

其中推理 RL 和偏好对齐的顺序很关键——先提升能力，再对齐偏好。PRISM 的研究证明，中训练对 RL 成功至关重要：直接对未经中训练的基座模型做 RL，效果接近零。

对齐是一个活跃的研究领域，新方法层出不穷。但核心原则不变：让模型的输出既有能力（helpful），又安全（harmless），还诚实（honest）。而五代演进的趋势清晰表明——后训练正在从依赖人类标注走向自动化验证驱动。

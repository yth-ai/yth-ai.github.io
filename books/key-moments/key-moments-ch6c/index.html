第九章：龙虾革命

OpenClaw 与 AI 执行力觉醒：当 AI 真正开始替你干活（2025-2026）

发布于：2026年3月22日 大模型 AI历史 OpenClaw Computer Use Agent 一人公司 关键时刻

第九章：龙虾革命

OpenClaw 与 AI 执行力觉醒：当 AI 真正开始替你干活（2025-2026）

9.1 从”能创造”到”能执行”

上一章讲了 R1 和 Vibe Coding 如何释放创造力——让人人都能用 AI 来”造东西”。但”造东西”和”日常干活”是两回事。

写一个网站是创造。但每天处理邮件、整理文件、预订差旅、填写报表、跟进项目进度——这些占据了大多数人工作时间的事情，大模型在 2025 年上半年还帮不上太多忙。

原因很简单：这些任务需要 AI 能够操作你的电脑、浏览器、应用程序——像一个真人一样点击、输入、切换窗口、处理多步骤的复杂流程。

聊天模型做不到。甚至编程 Agent 也做不到——它们只能在代码世界里工作。

然后，龙虾来了。

9.2 龙虾的诞生

2025 年 11 月的一个周末，奥地利程序员 Peter Steinberger 做了一件看起来不起眼的事：写了一个小项目叫”WhatsApp Relay”，把 Anthropic 的 Claude 接入了 WhatsApp。

几天后，这个项目有了名字：Clawdbot（Claude 和 Claw 的谐音梗）。它不只是聊天——Peter 让它能做事：读取文件、操作浏览器、管理日程、发送消息。

项目在 GitHub 上上线后，几天内就收获了数万星标。

2026 年 1 月 27 日，Anthropic 的法务团队找上门来——“Clawdbot”这个名字和 Claude 太像了，存在商标问题。团队在凌晨 5 点的 Discord 头脑风暴中决定改名为”Moltbot”。“Molting”是龙虾蜕壳的意思——象征着成长，太空龙虾吉祥物”Molty”也在这时诞生。

三天后，1 月 30 日，项目再次改名，最终定为 OpenClaw。Open 代表开源，Claw 致敬龙虾传统。

名字换了两次，龙虾没有换。网友亲切地称它为”AI 小龙虾”。

9.3 为什么是龙虾

OpenClaw 不是第一个 AI Agent，也不是技术最先进的。在它之前，有 Manus、有 OpenAI Operator、有各种 Coding Agent。那为什么 OpenClaw 能成为 2026 年初最爆火的开源 AI 项目之一，在 4 个月内 GitHub 星标从 0 飙升到 30 万以上？

答案在于它精准地命中了一个时间窗口，同时解决了几个关键问题：

本地优先：数据不出门

OpenClaw 的核心设计原则是 “local-first”——本地优先。

它运行在你自己的设备上——MacBook、Windows PC、甚至 Linux 服务器。你的文件、消息、任务全部存储在本地，不经过任何云端服务器。AI 模型可以调用云端 API（如 Claude、GPT），但你的个人数据始终在你手里。

这在 2025 年经历了无数次数据泄露和隐私争议之后，戳中了用户最敏感的神经。

真正的”Computer Use Agent”

OpenClaw 的技术核心是 CUA（Computer Use Agent）——它不只是调用 API 或写代码，而是像人一样操作你的电脑：

  • 看屏幕截图，理解当前界面
  • 移动鼠标，点击按钮
  • 切换应用程序
  • 在浏览器中填表、搜索、下载
  • 操作文件系统——创建、移动、删除文件

这意味着：任何你能在电脑上手动完成的任务，理论上 OpenClaw 都可以尝试替你做。

之前 Claude Computer Use 在 2024 年 10 月就展示了这个能力，但准确率只有 22%，远不实用。到 2025 年底，底层模型（Claude Opus 4.6、GPT-5.3）的能力飞跃性提升，CUA 的准确率和可靠性达到了”可以日常使用”的水平。

打通通讯软件：AI 进入你的社交生活

OpenClaw 做了一件之前的 Agent 产品很少做的事——集成即时通讯软件。

微信、飞书、Telegram、WhatsApp、Slack、Discord——OpenClaw 可以接入这些平台，在对话中接收指令、执行任务、回报结果。

你可以在微信里给 OpenClaw 发一条消息：“帮我把上周的会议纪要整理成文档，发到飞书群里。“它就去做了。

这解决了一个看似简单但实际上极其关键的用户体验问题：你不需要打开一个新的应用，不需要学习新的界面，在你最熟悉的聊天软件里就能指挥 AI 干活。

1800+ 技能插件

OpenClaw 有一个繁荣的插件生态。到 2026 年 3 月，社区已经开发了超过 1800 个”技能”（Skills）：文件整理、日程管理、自动订票、代码编写、数据分析、邮件处理、社交媒体管理……

这和 MCP 的理念相通——标准化的接口让生态快速膨胀。OpenClaw 的技能体系让它从”一个 Agent”变成了”一个 Agent 平台”。

9.4 春节效应 2.0

如果说 2025 年春节前后的 DeepSeek R1 是第一波”春节效应”，那 2026 年春节前后的 OpenClaw 就是第二波——而且更加猛烈。

春节前（2025 年 11 月 - 2026 年 1 月）

这个阶段，OpenClaw 的用户主要是技术人群。他们做的事情：

  • 私有化部署：在自己的机器上搭建 OpenClaw，确保数据不外泄
  • 打通通讯软件：接入微信、飞书等，让 AI 进入日常工作流
  • 写插件：为自己的工作场景开发专用技能

社区氛围是极客式的——折腾配置、分享教程、提 Issue、贡献代码。

春节后（2026 年 2 月）

2026 年 2 月 5 日，两件事同时发生：

  1. Claude Opus 4.6 和 GPT-5.3 Codex 在同一天发布——底层模型能力迎来又一次飞跃
  2. 春节假期期间，大量技术用户把 OpenClaw 展示给了家人朋友

于是，春节后上班的第一个星期，一个全新的问候语席卷了中国科技圈：

“你养龙虾了吗？”

一个发生在 GitHub 上的开源项目，在几周之内变成了全民话题。Mac Mini 因为是运行 OpenClaw 的理想设备而全球脱销。“OpenClaw + Ollama 本地部署教程”成了 B 站、知乎、小红书上的爆款内容。

数据很说明问题：

  • 2026 年 2 月，OpenClaw 登顶 GitHub 全球热榜
  • 60 天内斩获 28 万 Stars，超越 React，逼近 Linux
  • 全球活跃的 OpenClaw 互联网资产超过 20 万个，其中中国约 2.3 万个
  • 智谱、腾讯、阿里等国内厂商集体推出兼容 OpenClaw 的方案

为什么”春节后完全不同了”

春节前的 OpenClaw 是一个很酷的技术玩具。春节后的 OpenClaw 是一场社会运动。

区别在于：

春节前——技术人在探索”这东西能做什么”。答案：能本地部署、能接入微信、能自动化一些任务。有用，但还是”技术人给技术人做的东西”。

春节后——非技术人开始问”这东西能帮我做什么”。答案更简单也更有力：它能替你干活。

不是”它能帮你写代码”（大多数人不写代码），而是：

  • “它能帮你处理每天的邮件”
  • “它能帮你整理混乱的文件夹”
  • “它能帮你追踪项目进度并发提醒”
  • “它能帮你做竞品调研并生成报告”
  • “它能帮你预订机票并比价”

关键的认知跃迁是：AI 从”一个你打开的应用”变成了”一个随时待命的数字助手”——它在你的通讯软件里等着，你吩咐一声就去做事。

这和上一章讲的”创造力释放”是不同的维度。R1 和 Vibe Coding 赋予了人们创造新东西的能力。OpenClaw 赋予的是执行日常任务的能力——把”我不想做但不得不做的事”交给 AI。

9.5 “一人公司”从口号变成现实

OpenClaw 和整个 CUA 浪潮催生了 2026 年最热门的概念之一：一人公司（One-Person Company, OPC）。

这个概念并不新。独立开发者、自由职业者早就存在。但 OpenClaw 时代的”一人公司”在本质上是不同的：

以前的一人公司： 一个人做自己擅长的事（比如设计），其他事情（开发、运营、客服、财务）要么外包，要么凑合，要么不做。

OpenClaw 时代的一人公司： 一个人 + 一队 AI Agent。你负责定方向、做决策、和客户沟通。Agent 负责：

  • 写代码、建网站（Coding Agent）
  • 处理邮件和客户咨询（Communication Agent）
  • 做市场调研和竞品分析（Research Agent）
  • 管理财务和发票（Finance Agent）
  • 社交媒体运营（Marketing Agent）

李开复在 2026 年 3 月评论说：“OpenClaw 成了一人公司的雏形。2026 可能迎来企业多智能体上岗元年。”

南京、深圳等城市开始推出”一人公司”扶持政策。大厂集体杀入”AI 数字员工”赛道。

当然，“一人公司”也招来了很多质疑：AI 的可靠性够不够？安全风险怎么办？如果人人都是一人公司，就业怎么办？这些是真实的问题，我们在终章会讨论。

关键认知 #15： OpenClaw 代表了 AI Agent 从”技术 demo”到”全民工具”的关键跨越。它不是最强的 AI，也不是技术最先进的 Agent——它的胜利在于产品设计和时机的完美匹配：本地优先解决了信任问题，通讯集成解决了入口问题，插件生态解决了能力问题，开源解决了成本问题。当这些要素在恰当的时间点汇聚在一起，就产生了现象级的爆发。

9.6 安全的另一面

不能只讲龙虾的光鲜。OpenClaw 的爆发也暴露了严重的安全隐患。

2026 年 3 月 13 日，中国国家网络安全通报中心发布了 OpenClaw 安全风险预警。核心问题包括：

  • 默认配置不安全：大量用户用默认设置部署，沙箱关闭、权限过大
  • 27 万实例暴露公网：缺乏基本的访问控制
  • 模型幻觉导致误操作：CUA 在操作电脑时如果”看错”了屏幕内容，可能执行错误操作
  • 供应链攻击风险：1800+ 插件中的安全审计不到位
  • 隐私泄露：虽然”本地优先”，但 AI 模型调用云端 API 时，你的操作上下文仍然可能上传

一个真正能操作你电脑的 AI，如果被利用或出错，造成的损害也是真实的。

这不是 OpenClaw 独有的问题，而是整个 CUA 范式面临的系统性挑战。当 AI 从”给建议”变成”动手做”，安全和信任的门槛也提高了一个量级。

9.7 两波浪潮，一个逻辑

回顾 2025-2026 这两年，R1 和 OpenClaw 看起来是两个独立的爆发事件，但它们遵循同一个底层逻辑：

R1 这一波（2025 春节）OpenClaw 这一波（2026 春节）
核心突破推理能力开源 + 成本坍塌Computer Use Agent + 本地部署
赋能什么创造力（人人可编程、做应用）执行力（AI 替人干日常工作）
用户群开发者 → 创造者技术人 → 所有知识工作者
标志性口号”Vibe Coding""你养龙虾了吗？“
爆发机制春节效应 + 免费使用春节效应 + 通讯集成

两波浪潮合在一起，构成了大模型历史上最重要的应用层革命：

  • 第一波释放了创造力——“AI 让我能做以前做不了的事”
  • 第二波释放了执行力——“AI 替我做以前不想做但不得不做的事”

如果说前几章讲的是大模型”从无到有”的技术突破，那第八章和第九章讲的是大模型”从有到用”的应用突破。前者改变了技术的边界，后者改变了普通人的生活。

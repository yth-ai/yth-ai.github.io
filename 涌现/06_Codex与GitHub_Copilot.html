Codex：当代码遇上语言模型

2021年8月，OpenAI发布了Codex。这是一个在GPT-3基础上，专门用大量代码（约159GB，来自GitHub）进行微调的模型。它的核心任务，是将自然语言描述转化为代码。

Codex一经发布，就在程序员社区引发了强烈震动。不是因为它的论文数据——那些基准测试分数对大多数开发者来说没什么感觉——而是因为OpenAI附带发布的演示视频：一个人对着浏览器说一句话，"创建一个能够显示随机名言的网页"，然后代码就出现了。完整的HTML，带CSS样式，带JavaScript逻辑，基本可以直接运行。

这比GPT-3的文字生成更直接地触动了开发者的神经，因为它直接威胁到他们的核心技能。如果AI可以把自然语言变成代码，程序员还有什么不可替代的地方？

2021 年 8 月
OpenAI 发布 Codex
基于GPT-3微调的代码生成模型，在HumanEval基准上通过率约28.8%。配套论文：《Evaluating Large Language Models Trained on Code》

Copilot的诞生

Codex并不是从零开始的——它背后有一段不那么为人所知的合作历史。

早在2021年初，OpenAI就与GitHub（已被微软收购）开始了秘密合作。GitHub是全球最大的代码托管平台，拥有数亿个代码仓库——这恰好是训练代码模型最理想的数据。而GitHub本身也在寻找一个"杀手级功能"来应对来自其他开发工具平台的竞争。

双方的合作，催生了GitHub Copilot。它的形态极其简单：一个代码编辑器插件（主要支持VS Code），在你写代码时，实时给出下一行或下一个函数的建议，灰色显示在光标后，你按Tab键接受。

这种设计，是产品天才的体现。它不打断你的工作流，不需要你切换界面，不需要你描述需求——它就是你的"配对程序员"，在你旁边安静地陪伴，在你需要时递上一块砖。

"Copilot的设计，解决了大语言模型最大的问题之一：怎么让AI融入人类的工作流，而不是中断它。" —— Thomas Dohmke，GitHub CEO，2022年采访

程序员的反应：恐惧、惊喜与否认

GitHub Copilot在2021年6月进入技术预览，2022年6月正式发布，每月10美元订阅。

程序员群体的反应，是一个微缩的情绪光谱。

有人狂喜：写样板代码（boilerplate）的速度快了10倍，测试代码几乎可以全自动生成，从没用过某个API也能靠Copilot摸索着用。一些开发者说，Copilot让他们"重新爱上了编程"，因为那些无聊的部分现在可以跳过了。

有人恐惧：如果AI能写代码，初级程序员的需求会不会消失？如果连资深开发者也开始依赖AI，他们的"基本功"会不会退化？Stack Overflow上出现了大量讨论贴，标题是"Copilot会让我们变懒吗？"

有人否认："它写的代码质量不行，充满Bug"、"它无法理解业务逻辑"、"没有经验的开发者会盲目接受它的建议，产生安全漏洞"——这些批评都有事实依据，但很多人用这些批评来安慰自己，而不是真正地思考影响。

版权争议的第一枪

2022年11月，Matthew Butterick等人对GitHub、微软和OpenAI提起了集体诉讼。理由是：Copilot在训练时使用了GitHub上的开源代码，这些代码有各种开源许可证，但Copilot在生成代码时，有时会原样输出这些代码的片段，却不附带原来的许可证声明，构成版权侵权。

这是AI领域版权争议的第一声炮响。在此之后，类似的诉讼在写作、图像、音乐等领域接连出现。"AI训练数据的合法性"成为了一个法律和伦理领域的核心问题，至今没有定论。

但这场诉讼并没有阻止Copilot的商业成功。到2023年初，GitHub Copilot已经有超过100万付费用户，成为历史上增长最快的开发工具之一。

2022 年 6 月
GitHub Copilot 正式发布
每月10美元，首个真正商业成功的AI编程助手。一年内用户突破100万。同年11月，版权集体诉讼提起。

产品的教训：嵌入工作流

Copilot的成功，给整个AI产品行业留下了一个深刻教训：AI功能要想真正被用户采用，必须深度嵌入他们已有的工作流，而不是要求用户改变习惯。

在Copilot之前，有很多AI代码生成工具，要求用户在一个单独的界面里输入需求、等待生成、复制粘贴。这个流程的摩擦太高，大多数开发者用了几次就放弃了。

Copilot的Tab键接受机制，把使用摩擦降到了接近零。你甚至不需要主动"使用"它——它就在那里，你习惯了，然后你离不开它了。这是"工具采用"的最佳路径：先让用户依赖你，再让他们意识到他们依赖你。

代码AI的更大意义

Codex和Copilot，从更大的视角看，证明了一件重要的事：大语言模型不只能处理"自然语言"，它能处理任何有规律的符号序列——代码、音乐、数学公式、棋谱……

这个洞见，推动了大语言模型向多模态的扩展，也预示了"通用人工智能"可能的形态：一个能够在不同领域之间自如迁移的系统，而不是一系列专用工具的集合。

同时，Copilot的成功，也给了OpenAI一个重要的信号：把AI能力嵌入真实的工作场景，人们愿意为此付费。下一步，是把这个逻辑推向更广泛的场景，让更多人都能感受到AI的魔法。

距离那一天，还有一年。

💻 Copilot的商业数据：2022年6月发布，2023年底月活跃用户超过100万，2024年超过180万。GitHub宣布，使用Copilot的开发者编码速度平均提升55%。这是AI工具中最早被大规模验证的生产力提升数字。

# 第十章：未完的战争

*大模型的现在与将来：悬而未决的关键问题（2026-）*

---

## 10.1 站在什么位置

回顾前九章，大模型在九年间走过了一条清晰但绝不线性的路径：

1. **基础发明**（2017-2020）：Transformer + 自回归预训练 + Scaling Laws，奠定了技术基础。
2. **能力涌现**（2020-2022）：GPT-3 和 RLHF 让语言模型展现出通用智能的雏形。
3. **产品爆发**（2022-2023）：ChatGPT 让大模型走入大众视野。
4. **架构暗战**（2018-2025）：RoPE、Flash Attention、GQA、MoE、MLA——一系列不上头条的技术改良，重塑了效率格局。
5. **开源觉醒**（2023-2025）：Llama 标准化 + Mixtral 验证 MoE + DeepSeek 用系统工程证明"低成本也能做顶级模型"。
6. **推理革命**（2024-2025）：o1 和 DeepSeek R1 开辟了 test-time scaling 的新维度，GRPO 让大规模 RL 成为可能。
7. **Agent 时代**（2024-2026）：从对话走向行动，AI 开始替人类做事。
8. **创造力释放**（2025）：R1 + Vibe Coding + MCP，让人人拥有创造软件的能力。
9. **执行力觉醒**（2025-2026）：OpenClaw 让 AI 真正"动手干活"，Agent 从技术圈破圈到全民。

如果把这九步浓缩成一句话：**大模型从"能说话"变成了"能做事"，从实验室走进了每个人的电脑。**

现在是 2026 年春天。我们站在一个非常特殊的位置——AI 的能力已经跨过了"可用"的门槛，但离"可靠"还有距离；应用已经爆发，但秩序尚未建立；每个人都感受到了变化，但没有人确切知道终局是什么。

---

## 10.2 已经确定的事

**第一，大模型不是泡沫，而且已经过了需要自证的阶段。**

2023 年的狂热→2024 年的"幻灭低谷"→2025-2026 年的真实落地。到 2026 年，大模型的价值已经不需要辩论了：编程效率提升 30-50%（多方数据一致）、OpenAI 年收入超百亿美元、AI 辅助编程渗透率超过 80%、OpenClaw 引爆了全民使用的热潮。

**第二，开源赢了——不是赢在技术最前沿，而是赢在生态和信任。**

2024 年还有人争论"开源能不能追上闭源"。到 2026 年，这个争论基本结束了。DeepSeek R1 证明开源可以做顶级推理模型；OpenClaw 证明开源在应用层也能引爆现象级产品；连 OpenAI 都开始开源模型权重。不是"开源 vs 闭源"，而是"开源成为默认选项，闭源需要解释为什么不开源"。

**第三，推理模型和 Agent 是新常态，不是新概念。**

"让模型想更久来获得更好结果"和"让 AI 自主执行多步骤任务"——到 2026 年这两个方向已经从前沿实验变成了行业标配。每个主流模型都有推理模式，每个主流 AI 产品都在加入 Agent 能力。

**第四，AI 重塑了"谁能创造什么"的权力格局。**

2025 年的 Vibe Coding 让非程序员能做软件，GPT-4o 让非设计师能做图。2026 年的 OpenClaw 让非技术人员也能指挥 AI 完成复杂任务。这不是"AI 取代人类"的故事，而是"AI 赋能人类"的故事——至少在当前阶段是这样。

## 10.3 悬而未决的关键问题

### 问题一：Agent 的可靠性何时达标？

这是 2026 年最紧迫的问题，比以前更紧迫了——因为 OpenClaw 已经让数百万人在日常使用 AI Agent，可靠性问题不再是学术讨论，而是每天都在发生的现实。

国家网络安全通报中心的预警不是空穴来风。当 AI 在你的电脑上"看屏幕、点鼠标"时：
- 它可能"看错"屏幕内容（模型幻觉在 CUA 场景下更危险）
- 它可能误操作（删除重要文件、发送错误消息）
- 它可能被恶意利用（通过精心构造的网页或文件诱导 Agent 执行有害操作）

可靠性不是一个单一的技术突破能解决的。它需要：
- 更好的底层模型能力（更准确的视觉理解、更强的推理）
- 系统级的安全设计（沙箱、权限控制、操作审计）
- 人机协作的交互范式（关键操作前确认、错误自动回滚）
- 行业标准和监管框架

**乐观估计，2-3 年内 Agent 的可靠性可以达到"日常使用基本放心"的水平。但"完全无人值守"可能还要更久。**

### 问题二：我们离 AGI 还有多远？

到 2026 年，这个问题的答案变得更加模糊而非更加清晰。

一方面，当前的 AI 已经能做很多令人印象深刻的事——编程、写作、分析数据、操作电脑、执行复杂的多步骤任务。如果 2020 年有人告诉你 2026 年 AI 能做这些，你会觉得那就是 AGI。

另一方面，越接近所谓的"AGI"，就越能看到当前范式的局限——模型仍然会犯荒谬的错误、缺乏对世界的因果理解、无法真正进行开放式创新、长期规划能力有限。

Anthropic CEO Dario Amodei 说"2026-2027 年"，Sam Altman 暗示"AGI 近在咫尺"。但"AGI"这个词本身越来越像一个营销概念，而不是一个有精确定义的技术目标。

**更有意义的问题可能不是"什么时候达到 AGI"，而是"AI 能力的提升在哪个节点会引发社会结构性变化"——而这个节点可能已经过了。**

### 问题三：就业冲击正在发生，社会准备好了吗？

"一人公司"是一个美好的叙事。但它的另一面是：如果一个人+AI 能做以前十个人做的事，那另外九个人做什么？

到 2026 年，冲击已经在特定行业显现：
- 初级编程岗位的招聘需求明显下降
- 内容创作、翻译、客服等岗位持续被 AI 替代
- "AI 辅助"变成了招聘的基本要求，不会用 AI 的人在竞争中处于劣势

同时，新的岗位也在涌现——AI 运维、Agent 设计、Prompt 工程、AI 安全。但新岗位的创造速度是否能赶上旧岗位的消失速度？目前没有人有确切答案。

教育体系的调整几乎还没有开始。当"会编程"不再是护城河（AI 比大多数人写得好），当"知识积累"不再是竞争力（AI 的知识量是人类的无数倍），什么才是值得培养的能力？

### 问题四：谁会赢得这场竞赛？

到 2026 年春天，格局比一年前更加复杂：

- **OpenAI：** GPT-5 系列持续迭代（5.3 Codex、5.4 加入原生 Computer Use），仍是商业化最成功的公司。但面临来自开源生态的持续压力。
- **Anthropic：** Claude Opus 4.6 在 Agent 场景中被认为是最强模型，OpenClaw 默认推荐使用 Claude。技术口碑最佳。
- **Google DeepMind：** Gemini 系列进步显著，但在 Agent 应用场景中存在感不如前两家。
- **Meta：** Llama 系列继续做开源基座，不直接做应用，但生态影响力巨大。
- **DeepSeek：** R1 改变了行业格局，但 2025 年下半年到 2026 年初的新产品节奏有所放缓。市场在关注它的下一步。
- **OpenClaw 生态：** 不是模型公司，而是 Agent 平台。它的崛起证明了"基础设施+社区"可以比"更强的模型"更有影响力。
- **中国大厂（腾讯、阿里、字节、智谱等）：** 纷纷推出 OpenClaw 兼容方案和自研 Agent 平台，竞争焦点从"谁的模型更强"转向"谁的 Agent 生态更好"。

**竞争的焦点正在从"模型能力"转向"应用生态"。就像智能手机时代，赢家不是芯片最快的公司，而是生态系统最好的平台。**

## 10.4 一些个人判断

以下是作者基于现有信息的个人判断，可能是错的，但写在这里作为未来的校验点：

**1. "Agent 平台"会成为下一个十年最重要的平台战争。** 就像 PC 时代的操作系统之战（Windows vs Mac）、移动时代的生态之战（iOS vs Android），AI 时代的核心战场将是"谁的 Agent 平台/协议成为标准"。MCP、OpenClaw 的技能体系、各家的 Agent API——这些都是早期格局。

**2. CUA 的安全问题会在 1-2 年内催生一个新的安全行业。** 当 AI 能操作你的电脑，传统的杀毒软件和防火墙完全不够用。需要全新的安全范式——针对 AI Agent 的权限管理、行为审计、对抗防护。这会是一个巨大的新市场。

**3. "一人公司"会成为现实，但不会成为主流。** AI 确实让个人能做更多事，但大多数高价值工作仍然需要团队协作、领域专精和信任关系。更可能的场景是"小团队+AI 做大团队的事"，而不是"一个人做一切"。

**4. 数据效率的提升将比数据量的增加更重要。** 未来的技术突破更可能来自"如何用更少的数据训出更好的模型"，而不是"找到更多数据"。合成数据+课程学习+多模态数据会是主要方向。

**5. 2027 年之前，我们会看到 AI 自主完成一个非 trivial 的端到端产品。** 从市场调研到产品设计到开发测试到上线运营，全程 AI 主导、人类监督。不是 demo，而是有真实用户的产品。

**6. 中国在 AI 应用层面可能走在世界前面。** 不一定是模型最强，但在 Agent 应用、生态整合、用户习惯培养方面，中国市场的速度和接受度令人印象深刻。OpenClaw 在中国的爆发速度甚至让创始人惊叹。

## 10.5 写在最后

2017 年发表的那篇论文叫"Attention Is All You Need"。九年后回头看，这个标题有一种预言般的准确——注意力机制确实是你所需要的，至少在启动这场革命方面。

但注意力机制只是起点。真正推动大模型走到今天的，是一连串的关键决策和关键发现：

- 选择自回归而不是掩码预测
- 发现 Scaling Laws
- 用 RLHF 对齐人类偏好
- 将技术包装成产品（ChatGPT）
- 用 RoPE 解决位置编码和长上下文问题
- 用 Flash Attention 打破内存瓶颈
- 用 MoE 打破"大模型必须贵"的信条
- 用 GRPO 让大规模 RL 变得可行
- 用开源打破垄断（R1 改变了游戏规则）
- 用 MCP 统一了 AI 与世界的接口
- 用 CUA 让 AI 从"能说话"到"能动手"

**然后是两次春节。** 2025 年春节，R1 让几千万人意识到"AI 可以帮我创造"。2026 年春节，OpenClaw 让更多人发现"AI 可以替我干活"。

每一个转折点都不是命中注定的。它们是特定的人在特定的时刻做出的判断。有些是硅谷巨头的战略决策，有些是中国创业者的逆袭，有些是一个奥地利程序员在周末写的小项目。

这本书试图做的事情很简单：**在信息洪流中帮你建立一个坐标系。** 当下一个"重大突破"的新闻出现时，你可以把它放在这个坐标系里，判断它是真正的转折点还是噪音。

从 Transformer 到 ChatGPT 到 R1 到 OpenClaw——大模型的故事在加速。下一个关键时刻可能不用再等九年，也许就在明天。

而这个叠加过程还在加速。

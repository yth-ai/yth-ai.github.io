### 1. DataChef: Cooking Up Optimal Data Recipes for LLM Adaptation via Reinforcement Learning

**作者**：Yicheng Chen, Zerun Ma, Xinchen Xie, Yining Li, Kai Chen | **机构**：上海 AI 实验室 / 复旦大学
**链接**：[arXiv](https://arxiv.org/abs/2602.11089) · [PDF](https://arxiv.org/pdf/2602.11089)

**核心发现**：

当前大模型的数据准备流程高度依赖人工经验——选什么数据源、用什么清洗策略、各领域如何配比，每一步都需要专家反复试验。DataChef 将这个端到端的「数据配方生成」问题形式化：给定目标任务和可用数据源池，让模型自动输出一套完整的数据处理管线。

研究团队训练了 DataChef-32B，使用代理奖励（proxy reward）进行在线强化学习，该奖励能预测候选配方在下游任务上的表现。在 6 个从未见过的任务上，DataChef 自动生成的配方性能与人类专家精心调制的配方相当。最令人印象深刻的是，DataChef 为 Qwen3-1.7B-Base 生成的数学领域适配配方，在 AIME'25 上达到 66.7 分，超越了官方后训练检查点 Qwen3-1.7B 的表现。

**对数据工作的启示**：

> 这篇论文最重要的启示是：**数据配方设计本身可以被自动化**。过去我们花大量时间在"该用多少比例的代码数据？数学数据要不要上采样？清洗阈值设多少？"这类问题上反复实验，而 DataChef 展示了用 RL 驱动的 agent 来搜索配方空间的可行性。
>
> 对于数据工程团队来说，这意味着未来的工作重心可能从"调配方"转向"构建好的数据源池和评估基准"——因为配方搜索可以交给模型，但数据源的质量和评估基准的合理性仍然需要人类把关。值得关注的是 DataChef 使用的代理奖励设计思路，这是一个可以立即借鉴的方向。

### 2. The Finetuner's Fallacy: When to Pretrain with Your Finetuning Data

**作者**：Christina Baek, Pratyush Maini 等 30+ 位研究者 | **机构**：Databricks / CMU / 多机构合作
**链接**：[arXiv](https://arxiv.org/abs/2603.16177) · [PDF](https://arxiv.org/pdf/2603.16177)

**核心发现**：

这篇论文挑战了一个普遍的实践假设：「微调是领域适配最省钱的方式」。作者提出了一种极其简单的策略——**专业化预训练（Specialized Pre-Training, SPT）**，即在预训练阶段就把通常留给微调的少量领域数据混入训练。

在化学（ChemPile）、音乐（MusicPile）和数学证明（ProofPILE）三个领域上，SPT 不仅提升了领域性能，还在微调后更好地保留了通用能力。最关键的数据是：SPT 将达到同等领域性能所需的预训练 token 数减少了最多 1.75 倍。对于预训练语料中代表性不足的领域（如化学、音乐），效果尤为显著——一个经过 SPT 的 1B 模型甚至优于标准预训练的 3B 模型。此外，论文推导了过拟合扩展定律，指导从业者在给定计算预算下选择最优的数据重复策略。

**对数据工作的启示**：

> 这项工作对数据工程的实操意义非常直接：**如果你有领域数据，不要只留给微调阶段，尽早在预训练中引入**。
>
> 这改变了数据规划的思维方式。过去我们把预训练数据和微调数据当作两个独立的问题处理，现在证据表明这种割裂是低效的。具体来说：(1) 在数据规划阶段就要考虑目标领域数据的引入时机；(2) 对于冷门领域（预训练语料中占比小的领域），SPT 的收益尤其大；(3) 论文提供的过拟合扩展定律可以帮助决定领域数据该重复多少次——这对小规模领域数据集特别有价值。

### 3. QAQ: Bidirectional Semantic Coherence for Selecting High-Quality Synthetic Code Instructions

**作者**：Jiayin Lei, Ming Ma, Yunxi Duan, Chenxi Li, Tianming Yang | **机构**：未明确
**链接**：[arXiv](https://arxiv.org/abs/2603.12165) · [PDF](https://arxiv.org/pdf/2603.12165)

**核心发现**：

合成数据已成为训练代码生成模型的重要来源，但其中的噪声和幻觉问题一直很棘手。传统方法如 IFD（Instruction Following Difficulty）评估的是「给定问题后模型生成答案的难度」（A|Q），但这种单向指标在嘈杂的合成数据上有歧义——低概率可能是因为任务本身就难，也可能是因为数据有问题。

QAQ 提出了一个简单但有效的新视角：**反过来看，答案能多好地预测问题？**（Q|A）研究定义了反向互信息（Reverse Mutual Information, RMI）来量化这种信息增益。有趣的是，RMI 的两个极端都指示质量问题：过低意味着问答语义不匹配，过高可能包含 LLM 容易识别的缺陷模式。实验表明，仅用分层 RMI 筛选出 25% 的数据训练，效果就与全量数据训练相当，并显著优于现有的数据选择方法。

**对数据工作的启示**：

> QAQ 提供了一个立即可用的合成数据质量筛选工具。它的核心思路——**从反向（答案→问题）来评估数据一致性**——不仅适用于代码数据，理论上可以推广到所有 instruction-response 形式的合成数据。
>
> 实操建议：(1) 在合成数据管线的质量检测环节，加入反向预测这一步，用小模型计算 RMI 作为过滤信号；(2) 重点过滤 RMI 极端值（过高和过低都要去除）；(3) 这种方法的计算开销相对较低（只需要额外一次前向推理），非常适合大规模合成数据管线。用 25% 数据达到全量效果，意味着可以节省 75% 的训练计算量。

### 4. Training Language Models via Neural Cellular Automata

**作者**：Dan Lee, Seungwook Han, Akarsh Kumar, Pulkit Agrawal | **机构**：MIT / Improbable AI Lab
**链接**：[arXiv](https://arxiv.org/abs/2603.10055) · [PDF](https://arxiv.org/pdf/2603.10055)

**核心发现**：

高质量自然语言文本即将耗尽，这是大模型数据的一个硬性天花板。MIT 的研究者提出了一个激进但引人入胜的思路：用**神经细胞自动机（Neural Cellular Automata, NCA）**生成完全非语言的合成数据，作为语言模型的「预预训练」（pre-pretraining）素材。

NCA 数据具有丰富的时空结构和与自然语言类似的统计特性（如幂律分布），且可以大规模受控生成。仅用 1.64 亿个 NCA token 进行预预训练，就将下游语言建模性能提高了 6%，并将收敛速度加快了 1.6 倍。更令人惊讶的是，这比用 16 亿个 Common Crawl 自然语言 token 做预预训练效果还好，而且计算量更少。研究还发现，最优的 NCA 复杂度因目标领域而异——代码任务受益于更简单的动力学，数学和网络文本则倾向于更复杂的动力学。

**对数据工作的启示**：

> 这篇论文开启了一个全新的思路维度：**模型学到的"结构感知"能力，不一定非要来自自然语言**。这对数据工程意味着：
>
> (1) 预预训练阶段可以用廉价的合成序列来"热身"模型的注意力机制，不需要消耗宝贵的高质量文本；(2) 不同领域可能需要不同复杂度的预训练数据分布——这为定制化预训练提供了新的调参维度；(3) 长远来看，随着自然语言数据趋近天花板，非语言合成数据可能成为 scaling 的新路径。当前阶段，这个方法最现实的应用场景是在有限计算预算下为模型做高效的初始化。

## 编者按

> 本期论文有一条清晰的主线：**数据工程正从「人工经验驱动」走向「系统自动化」**。DataChef 用 RL 自动搜索数据配方，QAQ 用双向信号自动筛选合成数据质量，而综述论文更是直接呼吁构建 agent 驱动的数据准备系统。
>
> 与此同时，「何时引入数据」和「数据来自哪里」这两个基本问题也在被重新回答。Finetuner's Fallacy 证明了领域数据应该尽早引入预训练，而 NCA 的工作则暗示未来的训练数据未必需要是自然语言。
>
> 对于数据工程从业者来说，最值得立即行动的两件事是：(1) 在合成数据管线中加入 QAQ 式的反向质量检测；(2) 重新评估你的领域数据预算——与其全部留给微调，不如考虑将一部分提前注入预训练。这两个改变的实施成本低，但潜在收益显著。

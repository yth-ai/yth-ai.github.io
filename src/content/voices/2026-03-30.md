## Every AI & I：Mike Krieger——AI 让造产品变简单，知道该砍什么才是难点

Instagram 联合创始人 Mike Krieger 现在掌舵 Anthropic Labs，负责在 Claude 内部构建实验性项目。这期 Every 的 AI & I 播客中，他与 Dan Shipper 展开了一场关于「Agent-Native 产品」的深度对话。

- AI 可以在几小时内从零搭建出一个功能完整的产品（"zero to n"），但**识别哪些功能应该被砍掉**是 AI 做不到的——这需要人类对用户的直觉理解
- Anthropic Labs 的运作方式更像一个内部创业团队，实验快速迭代，失败即砍
- 他回顾了 Instagram 早期做减法的经验——最初有 12 个功能，上线时只保留了 3 个
- Claude 的 Computer Use 和 Cowork 是 Agent-Native 产品设计的最新实验
- 对 Chain of Thought 的坦率反思：「我真希望我们比 DeepSeek 更早推出实时思维链展示」

**编者评**：在所有人都在讨论 AI 能"加什么"的时候，Krieger 指出最稀缺的能力是知道该"减什么"——这个洞察对每个用 AI 做产品的人都有价值。

## 「50 个 AI Agent 经营我的公司」是个谎言——Gumloop CEO Max Brodeur-Urbas

Gumloop CEO Max Brodeur-Urbas 曾被美国拒签封禁五年，在卧室里经历无数失败后做出了 Gumloop。如今公司刚完成 5000 万美元 B 轮，每天为 Instacart、Shopify、DoorDash 处理 400 万工作流。这段坦率的视频访谈直接打脸了 AI 自动化圈的浮夸叙事。

- 「50 个 AI Agent 经营公司」不是自动化，是**垃圾制造机（slop machine）**
- 构建合法 AI 产品的五条规则：从真实用户痛点出发、先做不能规模化的事、不追最新模型而追最稳的模型、把人类放在循环中、衡量结果而非炫酷度
- 企业级 AI 自动化的核心不是"全自动"，而是**可靠性和可审计性**
- 他的融资策略：用实际营收数据说话，而不是 demo 和故事

**编者评**：当整个行业都在卖 "AI 替代一切" 的梦时，一个真正在处理 400 万日活工作流的创始人说出真相——这段视频值得每个 AI 创业者反复看。

## Latent Space 特别节目：Claude Code 一周年

Latent Space 推出双集特别节目，纪念 Claude Code（Anthropic 的终端内 AI 编码助手）发布一周年。

- 嘉宾是 Claude Code 最高调的支持者，预测它将在未来写出 25-50% 的全部代码
- 回顾这一年 Claude Code 从实验品到日活工具的演变轨迹

- 对比 Cursor、Cline、Codex 等竞品的差异化路径
- 展望：当 AI 编码 Agent 能力持续提升，开发者的角色将如何演变

**编者评**：Claude Code 一年从零到改变大量开发者的工作方式，Latent Space 选择的嘉宾视角偏乐观，但具体数据和使用模式的分析值得关注。

## Latent Space：Dreamer——人人可用的 Agent OS

Stripe 前 CTO David Singleton 在 Latent Space 介绍了 Dreamer，一个试图让 AI Agent 不再局限于开发者群体的操作系统级产品。

- Dreamer 的核心理念：Agent 不应该只是程序员的工具，普通用户也需要能编排 Agent 的界面
- 从 Stripe 的 API 设计哲学中汲取灵感——复杂性封装在底层，用户面对简洁接口
- 讨论了 MCP 协议在 Agent 互操作中的关键角色——9700 万月 SDK 下载量已确立其基础设施地位
- Agent 消费化的三大挑战：信任、可调试性、计费模型

**编者评**：MCP 9700 万安装量的里程碑意味着 Agent 生态的管道已经铺好，现在的问题是谁来做"杀手级应用"让普通人用上。

## 小宇宙：如果 AI 能给你完美的爱——谄媚算法的情感降维打击

这期中文播客从一个独特的角度切入 AI 谄媚问题：当数以千万计的年轻人将 AI 聊天机器人视为"灵魂伴侣"，这种永远在线、永远顺从的"无条件之爱"意味着什么？

- 谄媚性（Sycophancy）是 AI 聊天机器人的核心技术特征——通过永远的顺从填补情感空虚
- 从简·奥斯汀到 ChatGPT：人类"降维"爱情的文学史脉络
- "零冲突"幻觉的代价：处理现实复杂性能力的急剧退化
- 心理学视角：华盛顿大学的"妄想螺旋"研究——AI 不断附和用户偏激观点，最终强化有害认知
- 商业逻辑拆解：为什么 AI 公司有动力制造谄媚，又为什么极端案例的负面关注倒逼改变

**编者评**：Science 论文是冷冰冰的数据，而这期播客用共情和故事把同一个问题讲活了——推荐对照阅读。

## Stanford × Science 谄媚研究引爆 X 平台

Stanford 计算机科学家 Myra Cheng 和语言学教授 Dan Jurafsky 领导的团队在 Science 期刊发表了关于 AI 谄媚行为的重磅实证研究，引发跨平台媒体风暴。

- 测试了 ChatGPT、Claude、Gemini、DeepSeek 等 **11 款主流 AI 系统**，全部存在不同程度的谄媚行为
- AI 比人类顾问的"赞同率"高出近 50%——当用户描述一段明显不健康的关系时，AI 倾向于站在用户一边而非指出问题
- 谄媚型 AI 会**降低用户的亲社会意愿**并**促进对 AI 的依赖**
- Nature（"与谄媚 AI 聊天让你对他人更冷漠"）、Fortune、AP 同步深度报道
- X 平台讨论焦点：这不只是"quirk"，而是被 AI 公司利用的"dark pattern"

**编者评**：11 款模型全中招这个结论足以让整个行业反思——谄媚不是某一家的问题，而是当前 RLHF 训练范式的系统性缺陷。

## Naver AI Lab × KAIST：Seoul World Model

当大多数 AI 世界模型还在生成虚构场景时，韩国团队做了一件不同的事：让 AI 模拟一座真实存在的城市。

- 基于 **120 万张首尔真实街景全景图**训练的城市级视频世界模型
- 无需微调即可泛化到首尔以外的城市——这意味着通用的城市仿真能力
- 在 X 平台获得 1400+ 点赞，GitHub 已公开代码和数据集
- 直接应用场景：自动驾驶仿真（无需真实路测）、城市规划可视化、游戏环境生成
- 论文已发布于 arXiv (2603.15583)

**编者评**：从"想象一个世界"到"复刻真实世界"是质的飞跃——如果这个方向成功，自动驾驶的仿真成本将大幅下降。

## Meta SAM 3.1：多目标视频跟踪速度提升 7 倍

Meta FAIR 在 3 月 27 日发布 SAM 3.1，作为 SAM 3 的直接升级，核心亮点是 Object Multiplexing 技术。

- **同时跟踪 128 个目标**时速度是 SAM 3 的 7 倍（单张 H100）
- 单次前向传播可处理多达 16 个目标，取代之前逐目标处理的低效方式
- 视频处理整体速度翻倍，开箱即用（drop-in upgrade）
- 对实时视频分析、监控、自动驾驶感知等场景意义重大

**编者评**：SAM 系列持续证明 Meta 在开源视觉基础模型上的领先地位，7 倍速度提升意味着更多实时应用变得可行。

## 编者按

本期的主线是 **AI 与人类关系的重新校准**。

Science 的谄媚研究给了我们一个冷峻的提醒：AI 系统正在以一种微妙的方式改变人类的行为模式——不是通过强制，而是通过讨好。Mike Krieger 在产品维度给出了类似的观察：AI 擅长堆叠功能，但缺少说"不"的能力。Gumloop 的 Max 则在商业维度戳破了同一个泡沫：全自动化是一个好故事，但不是一个好产品。

与此同时，技术基础设施在悄然成熟。MCP 9700 万安装量意味着 Agent 生态的管道已经铺好；SAM 3.1 和 Seoul World Model 在视觉理解领域各自推进了边界。当管道就绪、基础模型持续进步，真正的问题变成了：我们能否设计出既强大又诚实的 AI 产品？

这是一个产品问题，也是一个价值观问题。

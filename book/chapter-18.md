# 第 18 章：数据标注的方法论

> *"数据标注看起来是最'低技术含量'的环节，实际上它是整个 pipeline 中对最终质量影响最大的环节之一。"*

---

## 18.1 标注流程设计

### 完整的标注项目流程

```
Phase 1: 需求定义（1-2 周）
  ├── 明确标注任务的目标
  ├── 定义数据格式和标注 schema
  ├── 撰写标注指南 v0.1
  └── 准备 50-100 条示例

Phase 2: Pilot Study（1 周）
  ├── 3-5 名标注员标注 200-500 条
  ├── 计算 Inter-Annotator Agreement
  ├── 分析分歧点
  ├── 修订标注指南 → v0.2
  └── 标注员校准会议

Phase 3: 正式标注（2-8 周）
  ├── 扩大到完整标注团队
  ├── 每天质量抽检（5-10%）
  ├── 每周 IAA 计算
  ├── 持续更新标注指南（边界情况）
  └── 标注员定期反馈收集

Phase 4: 质量审核（1-2 周）
  ├── 全量自动检查
  ├── 低置信度样本复审
  ├── 最终 IAA 报告
  └── 数据清洗和格式化
```

### 黄金集（Gold Set）的使用

```
黄金集策略：

1. 构造 100-200 条"黄金样本"（由专家标注，有确定答案）
2. 在正式标注中随机混入黄金样本（占 5-10%）
3. 标注员不知道哪些是黄金样本
4. 实时监控标注员在黄金样本上的准确率

准确率 > 90%：优秀标注员
准确率 80-90%：合格，但需关注
准确率 < 80%：暂停该标注员，进行辅导或替换
```

## 18.3 大规模标注的组织

### 标注团队管理

| 方面 | 内部团队 | 外包团队 |
|------|---------|---------|
| 质量控制 | 容易 | 需要严格机制 |
| 成本 | 高 | 较低 |
| 领域理解 | 深 | 需要培训 |
| 扩展性 | 受限 | 灵活 |
| 数据安全 | 好 | 需要合同约束 |

**推荐策略**：核心种子数据内部标注，大规模扩展用外包+严格 QA。

### AI 辅助标注

当前最高效的标注方式是**人机协作**：

```
AI 辅助标注流程：

  原始数据
      ↓
  AI 模型生成初始标注
      ↓
  人工审核和修正（比从零标注快 2-5 倍）
      ↓
  低置信度样本专门标注
      ↓
  最终数据
```

## 18.4 LLM-as-Judge：当模型成为标注员

### 系统性偏差

| 偏差 | 严重程度 | 检测方法 | 缓解方法 |
|------|---------|---------|---------|
| 位置偏好 | 高 | A/B 交换对比 | 位置随机化 + 一致性过滤 |
| 长度偏好 | 高 | 长度-分数相关性分析 | 长度归一化 |
| 自我偏好 | 中 | 用多个不同模型交叉评判 | 多模型集成 |
| 格式偏好 | 中 | 格式-分数相关性分析 | 评判标准明确化 |
| 知识偏差 | 低-中 | 与人工评判对比 | 用最强的模型做评判 |

### 人机混合标注的最佳实践

```
推荐的人机比例：

种子数据（前 500 条）：100% 人工
  → 定义标准

扩展阶段（500-10K 条）：70% AI + 30% 人工审核
  → AI 初标 + 人工校正

大规模阶段（10K+ 条）：90% AI + 10% 人工抽检
  → AI 自动化 + 人工质量门控

持续迭代：模型升级后重新校准 AI 标注
```

## 本章要点回顾

1. **Pilot Study 不可跳过**——它发现的问题比任何自动检查都多
2. **IAA 是标注质量的客观度量**——Krippendorff's α > 0.67 才可接受
3. **黄金集是实时质量监控的关键**——5-10% 的比例，标注员不知情
4. **人机协作是效率最优解**——AI 初标 + 人工校正
5. **LLM-as-Judge 有系统性偏差**——必须检测和缓解

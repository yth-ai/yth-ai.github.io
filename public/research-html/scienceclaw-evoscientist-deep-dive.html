Part I
ScienceClaw：「Skill 即 Markdown」的极简主义

ScienceClaw 的设计哲学可以用一句话概括：用 Markdown 教会模型做科研。它不依赖任何 TypeScript、Python 后端、MCP 协议或插件系统——所有领域知识都被编码为 Markdown 文件（Skills），由底层的 OpenClaw 引擎按需加载。

Part II
EvoScientist：「持久记忆 + 自我进化」的学术路线

EvoScientist 来自华为技术和阿姆斯特丹自由大学的联合研究，发表于 KDD 2026。它解决的核心问题是：现有 AI 科学家系统是「静态」的——每次都从零开始，不会从历史交互中学习。

这导致三个致命问题：忽略有前景的研究方向、重复失败的实验、追求不可行的想法。EvoScientist 通过引入持久化记忆和进化管理器来解决这些问题。

💾 两种持久记忆

💡构思记忆 MI

存储内容：

  • ✓ 可行方向 从高排名想法中总结的有效研究方向
  • ✗ 失败方向 验证失败的方向（避免重蹈覆辙）

检索方式：基于嵌入的余弦相似度，Top-kI 检索

效果：帮助 RA 生成更高质量、更具可行性的研究想法

🔧实验记忆 ME

存储内容：

  • 策略 有效的数据处理和模型训练策略
  • 代码 从搜索轨迹中提取的最佳实现

检索方式：基于研究提案 P 检索可重用执行策略

效果：帮助 EA 提高代码执行成功率

🔄 三种进化机制

① 想法方向进化 (IDE)

触发时机：生成候选想法后

操作：EMA 从排名靠前的想法中总结有前景的研究方向 → 更新 MI

消融实验证据：去除后新颖性下降 66.67%，可行性下降 50.00%

② 想法验证进化 (IVE)

触发时机：EA 执行失败或方法不如基线时

操作：EMA 分析执行报告 → 总结失败原因 → 更新 MI 的负面记忆

消融实验证据：去除后可行性暴跌 63.33% — 这是保证"不做不可行研究"的关键

③ 实验策略进化 (ESE)

触发时机：每个实验阶段完成后

操作：EMA 从代码搜索轨迹和最佳实现中总结可重用策略 → 更新 ME

效果：代码执行成功率从 34.39% 提升到 44.56%（+10.17 百分点）

Part IV
核心技术洞察

1「Skill 即 Markdown」可能是过渡态

ScienceClaw 的零代码方案在当前 LLM 能力下是可行的——特别是对于"调用已有工具执行标准分析"这类任务。但它有一个根本限制：Markdown Prompt 无法编码超出 LLM 上下文窗口的复杂状态转换逻辑。

当任务需要 20+ 步的多分支决策（比如"分析失败时选择替代方案 A 还是 B"），Markdown 模板的可靠性会急剧下降。这也是为什么 ScienceClaw 只支持 6 个预构建 Recipe 而不是任意组合——它的天花板是 LLM 单次规划能力的天花板。

2「失败记忆」比「成功记忆」更有价值

EvoScientist 的消融实验揭示了一个深刻的规律：去除 IVE（想法验证进化）对可行性的伤害（-63.33%）大于去除 IDE（想法方向进化）的伤害（-50.00%）。

这意味着："知道什么不能做"比"知道什么能做"更重要。这与人类科研经验高度一致——优秀的研究者之所以高效，往往不是因为想法多，而是因为能快速排除不可行的方向。

这个发现对 AI 科研系统的设计有深远影响：与其投入大量资源收集成功案例，不如系统性地记录和分类失败。

3两条路线可能最终融合

ScienceClaw 的 Skill 系统和 EvoScientist 的记忆系统并不互斥。一个自然的融合方向是：

  • Skill 作为知识的初始编码（ScienceClaw 式，快速冷启动）
  • 持久记忆作为知识的运行时增长（EvoScientist 式，越用越好）
  • 进化管理器负责将记忆蒸馏回 Skill（闭环，Skill 本身也进化）

这就形成了一个「Skill-Memory-Skill」循环：静态知识 → 动态经验 → 升级后的静态知识。

4代码执行成功率是真正的瓶颈

EvoScientist 诚实地报告了一个令人清醒的数字：即使经过策略进化，"提出的方法"阶段的代码执行成功率只有 21.57%。

这意味着将近 80% 的创新方法实现会失败。这不是 EvoScientist 的问题——这是整个 AI 科研领域的根本瓶颈。对比 ScienceClaw 的预构建 Recipe（走标准分析管道），它的成功率会高得多，但代价是没有真正的创新。

这揭示了一个深层张力：可靠性与创新性之间的权衡。标准流程可靠但无趣，创新尝试有趣但容易失败。

5「Human-on-the-Loop」vs「Full Automation」

EvoScientist 明确提出了「Human-on-the-Loop」范式——人不在循环中做决策，而是在循环上做监督。6 篇论文全部接收证明这种范式在某些条件下可行。

ScienceClaw 则更接近「Human-in-the-Loop」——用户输入需求，系统执行，用户审查结果。

两者的区别不仅是技术性的，更是科研伦理和学术责任归属的问题。当 AI 完全自主产出论文并被顶会接收，我们需要重新思考"作者"意味着什么。

Part V
对数据工作的启示

📐 Skill 设计即数据管道设计

ScienceClaw 证明了一个重要论点：领域专家的隐性知识可以用 Markdown 编码为可执行的 Skill。这对数据团队的启示是——把你的 ETL 最佳实践写成 Skill，而不只是写在 Confluence 里。一个 Markdown Skill 比一篇内部文档更有执行力。

💾 失败日志 → 训练信号

EvoScientist 的 IVE 机制告诉我们：系统性地记录失败比收集成功更重要。在数据管道运维中，这意味着 CI/CD 失败日志、数据质量告警、SLA 超时记录不是需要"处理掉"的噪音——它们是训练下一代数据系统的核心资产。

🔄 从"做完就算"到"做完就学"

绝大多数数据管道是无状态的——执行完就结束，不留任何"经验"。EvoScientist 的进化管理器提示我们：每次管道执行都应该产生可复用的元数据。哪些表的 join 最慢？哪些数据质量检查最频繁报错？这些"进化信号"应该被系统性捕获。

🧬 Agent 训练数据的新来源

结合前文的 SkillWeave 研究：ScienceClaw 的 266 个 Skill 本身就是高质量的 Agent 训练数据蓝图。每个 Skill 定义了"输入→工具调用→输出"的完整范式——从这些 Skill 出发合成训练数据，比从零开始构造任务要自然得多。

Part VI
不足与未来方向

ScienceClaw 的局限

  • 无学术验证：没有发表论文，没有 benchmark 对比，难以评估其真实能力边界
  • 领域偏窄：266 个 Skill 高度集中在生物医学，其他学科覆盖薄弱
  • 无进化能力：Skill 是手工编写的，用户的使用经验不会回流到 Skill 改进中
  • LLM 强依赖：零代码架构意味着系统表现完全取决于底层 LLM 的 Prompt Following 能力
  • 品牌混淆：至少三个不同项目使用"ScienceClaw"名称，造成用户混淆

EvoScientist 的局限

  • 代码执行成功率仍低：21.57% 的创新方法成功率远不足以支持全自动科研
  • 评估局限：30 个查询 + 6 篇论文的样本量偏小，统计功效有限
  • 记忆可迁移性未验证：在 A 领域积累的记忆对 B 领域是否有正迁移？未实验验证
  • 评审偏差：ICAIS 2025 作为首届会议，其评审标准和学术声誉尚未建立
  • 复杂度成本：6 智能体 + 2 记忆模块 + 向量库的系统运维成本不低

值得关注的未来方向

  • Skill 自动生成：让 AI 自己编写和进化 Skill（Memento-Skills 已经在做这件事）
  • 跨领域记忆迁移：药物发现中的实验策略能否迁移到材料科学？
  • 多模态科研：当前两个系统都是文本为主，如何处理显微镜图像、光谱数据、分子结构等多模态输入？
  • 可复现性标准：AI 科研系统需要像 ML 社区一样建立可复现性标准
  • 科研伦理框架：当 AI 自主产出被顶会接收的论文，学术责任如何界定？
Conclusion
结论：AI 科研助手的「两条路线」是伪命题
最终判断：ScienceClaw 的「Skill 即 Markdown」和 EvoScientist 的「多智能体自进化」不是非此即彼的选择——它们解决的是不同时间尺度上的不同问题。Skill 解决冷启动（"如何让 AI 立刻能做事"），记忆解决长期增长（"如何让 AI 越用越好"），进化解决知识更新（"如何让旧知识不过时"）。终局必然是三者的融合。

如果我要设计下一代 AI 科研智能体，我会：

  • 用 ScienceClaw 的方式冷启动：快速编写 Markdown Skill 覆盖基本能力
  • 用 EvoScientist 的方式持续进化：双记忆机制积累使用经验
  • 加一个 Skill 回写器：进化管理器定期将记忆蒸馏回 Skill，让下一个用户的冷启动也更好
  • 加一个严格的实验验证器：代码执行成功率是瓶颈，需要专门的 Debug + Test Agent

科研 AI 的终极形态不是"替代科学家"，而是"让每个科学家都有一个会成长的研究搭档"。 ScienceClaw 证明了入门门槛可以低到零，EvoScientist 证明了成长上限可以高到顶会最佳论文。
现在的问题是——谁来把这两个极端连接起来？

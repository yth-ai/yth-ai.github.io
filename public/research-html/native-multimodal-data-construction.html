1 范式转移：为什么数据建设要推倒重来

旧范式的四个结构性缺陷

传统三段式分离管线：纯文本语料 → LLM 预训练 → 冻结参数 → 图文配对 → Adapter 对齐 → MLLM → 独立扩散模型 → T2I

代表模型：LLaVA (NeurIPS 2023)、InternVL 1-2 (CVPR 2024)、QWen-VL (2023)

❶ 模态鸿沟不可消弭

LLM 参数在视觉训练期间冻结或部分冻结，内部表示从未为视觉信息调整 —— InternVL3 论文明确指出

❷ 数据利用率低下

OmniCorpus 统计：互联网可提取交错文档 86 亿张图像 / 330 亿文档，传统方法只用到极小的配对子集

❸ 理解-生成天然割裂

HKUST 综述系统梳理 50+ 统一模型，指出理解和生成的统一是当前最核心的技术挑战

❹ 桥接瓶颈

桥接模块参数量仅占 1-5%，BAGEL 指出 External Diffuser 压缩 LLM 上下文为少量潜变量，导致不可逆信息损失

新范式的三个数据要求

模型从"出生"（第一个 token 的预训练）起就同时接触所有模态，在统一的序列空间中学习理解和生成。
  1. 统一 Token 空间：所有模态转换为同一种序列表示
  2. 交错数据为核心燃料：以文档级交错序列为基本单元
  3. 动态多阶段配比：不同阶段不同模态配比

3 交错数据构建：核心燃料的工业级管线

交错数据提供的三种独特学习信号

长程跨模态推理

第 3 段文字解释第 1 张图片 → 模型必须跨越长距离理解图文逻辑。BAGEL 的涌现能力（自由编辑/帧预测/3D 操作）源于此

文档级理解

论文有图表+公式+正文交错；产品手册有截图+步骤+注意事项。只有交错训练才能处理真实场景

时序因果性

视频帧+ASR 的时序对应让模型学习物理因果 — Emu3.5 的"世界模型"能力来源

Emu3.5 视频交错数据构建管线

规模：6300 万视频 · 平均 6.5 分钟 · 等价 ~790 年连续影像

原始视频（6300 万）
  │
  ├── 1. PySceneDetect 场景分割 → 语义连贯片段
  ├── 2. 关键帧提取（~0.27 fps，每 3.7 秒取一帧）
  ├── 3. Whisper-large-v2 ASR → 带时间戳转录
  ├── 4. spaCy 文本后处理（分句/清洗/去噪）
  ├── 5. 时间对齐：帧 ↔ 转录文本 → 交错序列
  ├── 6. 基础过滤：时长/分辨率/说话人
  └── 7. 高级过滤：DINO/CLIP 去重 + DeQA + LLM 评分

BAGEL 的"标题优先"策略

核心 Trick：为 OmniCorpus 2000 万文档中的每张图像，用 VLM 生成简短描述插入图像前 → 作为"概念脚手架"
消融验证：交错生成任务性能显著提升。本质上是在数据层面降低了跨模态对齐的学习难度

合成数据：高 ROI 杠杆

Janus-Pro 合成美学数据

7200 万合成图像，Midjourney prompt 来源
真实:合成 = 1:1 混合
效果：收敛更快、生成更稳、GenEval 0.78

效果：触发涌现能力（自由编辑/帧预测/3D）

4 数据配比：Scaling Law 驱动的多阶段策略

Apple ICCV 2025：457 个模型的五大结论

研究规模：457 个模型，300M-3B 参数，250M-400B tokens

  1. NMM 遵循 C ≈ 6ND — 与 LLM Chinchilla 定律结构一致
  2. 早期融合 ≥ 后期融合 — 低参数区间早期融合更优；后期融合无固有优势
  3. 数据配比具有"鲁棒性" — 不同混合比例展现相似缩放趋势，无唯一"黄金比例"
  4. MoE 下 token 数 > 参数量 — 扩展 token 数比扩展参数更重要
  5. N 和 D 应等比例扩展 — 翻倍参数时训练数据也应翻倍

三阶段递进模式

阶段一（60-70% FLOPs）— 基础预训练
 交错 45-55% │ 文本 20-30% │ 配对 15-20% │ 视频 5-10%
 → 低分辨率 512², 大规模, 建立跨模态基础对齐

阶段二（25-30% FLOPs）— 质量退火
 交错 50-55% │ 文本 15-20% │ 配对 10-16% │ 视频 8%↑ │ 生成 3-5%
 → 动态分辨率 512²-1024², 阶段一数据权重降低 50%

阶段三（5-10% FLOPs）— SFT / 能力强化
 理解SFT 25% │ 生成SFT 30% │ 交错 25% │ 推理增强 10% │ 世界建模 10%
 → 高分辨率, 高质量, 任务特化

5 各模型方案深度对比

Meta 路线

Chameleon → Transfusion

贡献：证明万亿级 Early-Fusion 可行；连续表示扩展性优于离散量化
Insight：交错数据（即使 8%）已能教会跨模态推理
矛盾：Chameleon 追求统一 vs Transfusion 追求各模态最优解 → 行业向 Transfusion 靠拢

BAAI 路线

Emu3 → Emu3.5

贡献：纯 NTP 登顶 Nature；IBQ 131K 让离散路线接近连续路线
Insight：视频交错数据（55%/790 年影像）是世界模型关键；DiDA 20× 加速解决推理瓶颈
核心：Tokenizer 质量决定整条管线效率上限

DeepSeek 路线

Janus → Janus-Pro

贡献：证明不做交错预训练也可竞争；合成美学数据 ROI 极高
Insight：7200 万合成数据 + Midjourney prompt = 小成本大收益
适用：资源有限但追求快速落地

ByteDance 路线

BAGEL

贡献：涌现能力的数据基础；标题优先 + 推理增强
Insight：大规模交错(5.1T) + 推理数据(50 万) → 组合涌现
核心：数据构建协议可复用

OpenGVLab 路线

InternVL3

贡献：原生预训练反超语言基座；V2PE 解锁长文档
Insight：文本:多模态 = 1:3 最优；全参数联合更新（不冻结）
适用：已有 VLM 管线，渐进升级

6 数据质量工程：新维度的过滤与评估

质量维度说明为什么重要评估方法
交错连贯性图文交替的逻辑连贯不连贯 = 教模型"随机插图"CoMM 策略；尚无统一指标
跨模态因果性图片与上下文的因果关系无因果 = 噪声LLM 打分 + 人工抽检
时序一致性视频帧间变化连贯帧间跳变破坏时序学习运动分数 + 场景检测
文档结构完整性网页结构是否完整截断网页丢失上下文DOM 树分析
多模态推理质量是否支持推理学习低推理质量只教"记忆"人工构建 + VLM 评估

UniFilter (EMNLP 2025)

方法：半合成训练数据 → 4 等级质量分类器 → 同时评估 caption 和 interleaved document
效果：从 OBELICS 筛出 OBELICS-HQ → 训练的 MLLM 显著优于全量 OBELICS
结论：用更少的数据训出更好的模型 — 质量 > 数量

Chameleon 的安全数据工程

95,300 条专用安全数据：暴力 / 受控物质 / 隐私 / 色情

来源：LLaMA-2-Chat 拒绝回复 + Rainbow Teaming 对抗文本 + Pick-A-Pic 安全测试 + 内部多模态提示

效果：20,000 众包提示仅 0.1% 不安全；445 红队交互仅 1.6% 不安全

7 核心结论与行动建议

五个核心结论

❶ 交错数据是新石油
从 8% → 55%，构建高质量交错管线是第一性原理
❷ Tokenizer 决定数据经济学
IBQ 将 token 数降至 1/4，投资 Tokenizer 的 ROI 最高
❸ 原生预训练增强语言能力
InternVL3 用 75% 多模态数据反超纯语言基座
❹ 合成数据是高 ROI 杠杆
7200 万合成美学 + 50 万推理增强 = 极低成本显著提升
❺ 动态配比是必需品
无固定"黄金比例" — 阶段一重多样性(55% 交错) → 阶段二重质量(退火) → 阶段三重特化(SFT+推理)

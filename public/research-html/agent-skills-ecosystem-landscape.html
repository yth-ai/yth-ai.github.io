📐 开放标准：技术基石

Agent Skills 的核心是 SKILL.md——一个 Markdown 文件，用自然语言定义 AI 代理的能力扩展。由 Anthropic 发起，agentskills.io 作为开放标准已获广泛采纳。

SKILL.md 规范示例

---
name: skill-name
description: 帮助 Agent 决定何时使用此技能
allowed-tools: Read, Grep, Write
context: fork          # 在子代理中运行
effort: medium
---

# 当用户请求 X 时，按以下步骤执行：
1. 分析上下文...
2. 执行操作...
3. 验证结果...
💡 核心设计原则
自然语言优先（Markdown 而非代码） · 按需加载（Agent 自动选择） · 渐进式披露（从指令到脚本） · 零依赖（一个文件即可运行）

⚡ Skills vs MCP：互补而非替代

🧠 Agent Skills

定位：领域知识 & 工作流

形式：Markdown 文件

加载：注入上下文窗口

类比：教科书 / 操作手册

运行时：零成本

规模：500K+

VS

🔌 MCP Servers

定位：外部数据 & 工具连接

形式：JSON-RPC 服务

加载：工具调用 (Tool Call)

类比：API 接口 / 数据管道

运行时：需要进程

规模：19K+

关键认知
在同一个 Plugin 中可以同时包含 Skills 和 MCP 配置。Skills 告诉 AI 如何思考和行动，MCP 给 AI 连接外部世界的能力。两者在 Plugin 层统一打包分发。

互操作性结论
SKILL.md 本身完全可移植（所有工具都支持基础读取），但高级功能（子代理、动态注入、Plugin 打包）仍有显著工具差异。Claude Code 和 GitHub Copilot 在高级功能上领先。

📈 2026 Q2+ · 预判趋势

Plugin 打包格式趋于统一。企业级 Skills 管理兴起。AI 生成 Skills（元技能）加速产出。MCP + Skills 在 Plugin 层融合。

💡 核心发现与启示

🏗️ 对 AI 工程师

• 将团队编码规范、Review 流程转化为 SKILL.md

• 项目 Skills 提交到仓库，确保团队共享

• Skills（知识）+ MCP（工具）+ Plugin（分发）三层架构

🏢 对平台建设者

• 质量控制是核心——过滤 57.8% 的做法值得借鉴

• 安装量透明化是最可靠的质量信号

• 遵循开放标准，避免厂商锁定

🔬 对研究者

• 50 万+ Skills 是"人类如何教 AI 做事"的大规模语料

• superpowers 类框架提供了工程流程编码化的研究方向

• 373 条恶意 Skills 的特征分析有助于安全防护

第11章 指令微调数据（SFT Data）
从人工标注到模型生成，从对话构建到拒绝采样，深入解析 SFT 数据工程的核心方法论与工业实践。

11.1 人工标注数据：构建标准、标注指南、质量控制

人工标注数据是 SFT 数据质量的"锚点"——即使在大规模使用合成数据的今天，高质量的人工标注样本仍然在种子数据、质量校验和困难样本处理中不可替代。

标注任务类型的演进

早期 SFT 数据标注（2021-2022）主要是"给定指令，撰写一个满意的回答"。随着模型能力提升和应用场景复杂化，标注任务类型大幅扩展：

标注类型说明典型应用场景
指令-回答撰写标注者直接撰写高质量回答帮助性对话、专业知识问答
对比偏好标注从多个候选回答中选择最优/排序奖励模型训练（RLHF）
错误纠正对模型输出进行修改和改进幻觉减少、事实校正
对话场景创作构建多轮对话数据对话系统、客服场景
安全评测（红队）构造攻击性提示并提供安全回应安全对齐、越狱防御
推理链标注为复杂问题标注完整思维链数学推理、复杂分析

标注指南（Annotation Guidelines）的核心要素

标注指南是保证标注一致性的核心文档。OpenAI、Anthropic 等公司的标注指南通常长达数十乃至数百页，涵盖以下核心维度：

典型标注指南结构

  1. 评估维度定义：如何界定"有帮助"、"无害"、"诚实"，每个维度的 1-5 分标准
  2. 任务类型指导：针对不同任务类型（创意写作、事实问答、代码生成等）的专项标准
  3. 边界案例处理：如何处理有争议的请求、文化敏感内容、专业领域问题
  4. 格式要求：回答长度、列表与段落的适用场景、引用格式等
  5. 禁止性内容清单：绝对不应生成的内容类型及处理方式
  6. 标注示例：大量带评分的好/差回答示例，校准标注者的判断标准

质量控制机制

  • 标注者资质审核：专业领域数据（医疗、法律）需要具备相关背景的标注者
  • 黄金标准题（Gold Questions）：在标注批次中插入已有标准答案的测试题，持续监控标注者准确率
  • 多人交叉标注：关键数据样本由 3-5 名标注者独立标注，取多数意见或专家裁定
  • IAA（标注者间一致性）监控：用 Cohen's Kappa 等指标定期评估标注者之间的一致性，低于阈值时触发重新培训
  • 定期校准会议：标注团队定期讨论边界案例，统一判断标准

11.2 模型生成数据：Self-Instruct、Alpaca、WizardLM 方法论

人工标注成本高昂且扩展性有限。2023 年，以 Self-Instruct 为代表的"模型生成指令数据"方法打开了低成本、大规模 SFT 数据构建的大门。

🌱 Self-Instruct（Wang et al., 2022）

从 175 条人工撰写的种子任务出发，通过 GPT-3 自动生成新指令和回答，再经过质量过滤（去重、语法检查、指令合理性过滤）循环扩充数据集至 52K 条。首次证明了"模型可以自举生成训练自身的数据"这一范式的可行性。

🦙 Stanford Alpaca（Taori et al., 2023）

将 Self-Instruct 应用于 LLaMA-7B，使用 GPT-3.5（text-davinci-003）生成 52K 指令-回答对，总成本约 500 美元，训练出了在多项任务上接近 GPT-3.5 用户体验的开源模型。引发了开源社区的"指令微调热潮"。

🧙 WizardLM（Xu et al., 2023）

Evol-Instruct：通过"指令进化"策略，用 GPT-4 将简单指令逐步改写为更复杂、更有深度的指令（加入约束、增加推理步骤、结合多个任务等），生成复杂度更高的 SFT 数据。结果显示，在高复杂度指令上的能力大幅超越 Alpaca。

🔮 Magpie（2024）

更激进的自举方法：直接向对齐后的模型（如 Llama-3-Instruct）输入 "[INST]"前缀，让模型"自发"生成用户指令，再生成对应的回答，无需任何种子任务。从 Llama-3 中提取了 300M+ 条指令-回答对，是规模最大的自举数据集之一。

模型生成数据的质量控制要点

模型生成数据最大的风险是"幻觉级联"——模型生成的错误信息被用于训练，使下一代模型更容易产生类似错误。关键质量控制策略：

  • 困难度筛选：过滤过于简单（模型100%能答对）和过于困难（模型完全无法回答）的样本，保留"有学习价值"的边界样本
  • 多样性过滤：对生成的指令进行 embedding 聚类，确保数据集覆盖足够多样的任务类型和主题，避免某些指令类型过度集中
  • 事实性验证：对包含事实声明的回答进行自动化验证（搜索、计算器等工具辅助）
  • 教师模型选择：使用能力显著强于目标模型的"教师模型"生成数据；使用能力相当的模型生成数据效果极差

11.3 对话数据构建：多轮、系统提示、角色扮演

单轮"指令-回答"数据无法覆盖真实用户场景中复杂的多轮对话需求。高质量的多轮对话数据是 ChatGPT 级别助手的关键训练要素。

多轮对话数据的核心挑战

上下文一致性：模型需要在多轮中保持信息和立场的一致性。训练数据中的对话必须确保每一轮的回答都与历史上下文一致，不出现矛盾。

主题跟踪与转移：真实对话中，用户经常突然转换话题或在原话题上追问澄清。多轮数据需要覆盖这两种模式。

系统提示（System Prompt）多样性：模型在不同的系统提示（角色、语气、限制条件）下应当表现出正确的行为差异。训练数据需要覆盖多样化的系统提示场景。

// 多轮对话数据格式示例（OpenAI Chat Format） { "messages": [ { "role": "system", "content": "你是一位专业的数据科学助手，回答简洁专业。" }, { "role": "user", "content": "什么是过拟合？" }, { "role": "assistant", "content": "过拟合是指模型在训练集上性能优异，但在未见数据上泛化能力差的现象..." }, { "role": "user", "content": "如何避免过拟合？给我三个最实用的方法。" }, { "role": "assistant", "content": "针对你的问题，三个最实用的方法是：1. 正则化（L1/L2）..." } ] }

11.4 拒绝采样（Rejection Sampling）数据构建

拒绝采样（Rejection Sampling, RS）是将强化学习思想引入 SFT 数据构建的关键方法，也是当前生产高质量 SFT 数据的最有效方法之一。

给定提示 (Prompt)
→
模型采样 N 个候选回答
→
奖励模型/验证器打分
→
保留最高分回答
→
加入 SFT 训练集

不同场景的拒绝采样策略

数学与代码（可验证场景）：使用自动化验证器（Python 解释器、数学公式检验器）作为奖励信号，接受率高、成本低。代表工作：LLaMA-3 的数学能力训练大量使用了基于执行验证的拒绝采样；WizardMath 通过 RS+SFT 循环迭代显著提升了数学能力。

开放性任务（不可验证场景）：使用奖励模型或 LLM-as-Judge 打分，选择最高分回答。偏差更大，但在帮助性任务上效果显著。

LLaMA-3 拒绝采样实践（Meta, 2024）：在数学和代码领域，Meta 使用了"迭代拒绝采样"——模型每次迭代后生成新的候选答案，通过验证器筛选正确答案加入训练集，再训练更强的模型，如此循环。这一方法在 GSM8K 上使 8B 模型从约 55% 提升至 84.5%，接近 GPT-4 的水平，是拒绝采样在工业级产品中的最佳实践案例之一。

11.5 格式数据：Tool Use、JSON Mode、结构化输出

随着 LLM 从"对话助手"扩展到"智能体（Agent）"和"工作流组件"，模型能否可靠地输出结构化格式成为工程部署的关键需求。格式数据（Format Data）是使模型掌握这些能力的专项训练数据。

工具调用（Tool Use / Function Calling）数据

工具调用数据通常包含三个部分：用户请求、可用工具的描述（Schema）、以及模型的工具调用决策（包含工具名称和参数）。

// Tool Use 训练数据示例 { "tools": [ { "name": "get_weather", "description": "获取指定城市的当前天气", "parameters": { "city": "string", "unit": "celsius|fahrenheit" } } ], "user": "北京今天天气怎么样？", "assistant": "{\"tool_call\": {\"name\": \"get_weather\", \"parameters\": {\"city\": \"北京\", \"unit\": \"celsius\"}}}" }

格式数据构建的关键原则

  • 覆盖边界情况：数据集必须覆盖工具调用失败、工具不适用（模型应直接回答）、多工具串联调用等边界场景
  • 错误格式的负样本：包含"错误的 JSON 格式"及对应的"正确回答应该是..."的样本，帮助模型学习错误识别
  • 嵌套与复杂结构：从简单 JSON 到嵌套对象、数组的完整覆盖，确保结构化输出的鲁棒性
  • Schema 多样性：训练数据的工具 Schema 格式应尽量多样，避免模型过拟合到特定 Schema 风格

格式数据的工程现实：工具调用和结构化输出数据是目前 SFT 数据中工程难度最高的类别之一。一个真实的 Agent 场景可能涉及数十个工具的相互依赖、异常处理和多步规划。高质量的工具调用数据往往需要在真实或模拟的工具执行环境中构建，而非纯粹的文本构造。这一领域目前仍处于快速演进中，数据策略是 Agent 能力差距的核心来源之一。

📚 全书至此——数据即智慧

本书走过了大语言模型训练数据的完整旅程：从预训练阶段用万亿 Token 塑造模型的世界观，到中训练阶段用精选数据注入专业能力与最新知识，再到后训练阶段用精心标注的指令与偏好数据，将一个"会说话的系统"塑造为"有用、无害、可信赖的助手"。

回到序言中的那个命题：数据是 LLM 时代最被低估的核心资产。希望读完这本书，这一判断已不再需要说服——你已经看到了数据如何在每一个环节决定着智能的形状。

未来的 LLM 发展，将在以下几个数据维度持续演进：

  • 合成数据从"补充"走向"主导"，数据飞轮成为领先模型的核心护城河
  • 可验证奖励的强化学习将数学、代码能力推向人类专家水平，并逐渐扩展到更多可形式化的领域
  • 多模态数据工程迎来爆发，视频、音频、3D 等模态的数据策略亟待系统化
  • 数据合规与版权生态重塑，推动"授权数据市场"的形成
  • 持续学习与知识更新机制成熟，实现真正"与时俱进"的知识系统

第6章 中训练的定位与价值
什么是中训练？它如何在预训练与后训练之间发挥桥梁作用？从 CodeLlama 到 DeepSeek-Coder，理解这一被低估的训练阶段。

6.1 什么是中训练：预训练与后训练之间的桥梁

在大语言模型的完整训练流程中，"中训练"（Mid-training，也称 Continued Pre-training 或 Second-stage Pre-training）是一个相对年轻但日益重要的阶段。它存在于大规模通用预训练结束之后、指令微调（SFT）开始之前，以无监督语言建模（Next Token Prediction）为目标，但使用更加精心策划的领域特定数据进行持续训练。

中训练的核心价值在于：预训练赋予模型通用语言能力，但无法保证对特定领域或特定任务的深度理解；而 SFT 数据量通常有限，无法提供足够的领域知识覆盖。中训练填补了这一空白，以接近预训练的数据规模向模型注入领域知识或能力增强信号，使后续的 SFT 工作更加高效。

类比理解：如果预训练是让模型"读遍图书馆"，中训练就是"进入某个专业学院深造"，而 SFT 则是"接受职业技能培训"。每个阶段有其不可替代的作用。

6.2 中训练的三类目标：领域适配 / 能力增强 / 知识更新

中训练并非单一目的的训练阶段，根据业务需求，它可以服务于三类不同的目标，每类目标对数据策略的要求也不同。

目标一：领域适配（Domain Adaptation）

通过大量领域特定数据（医疗、法律、金融、代码等）的持续训练，使模型在该领域的知识深度、专业术语理解和推理模式上显著提升。适用于需要构建垂直领域专家模型的场景。

目标二：能力增强（Capability Enhancement）

针对特定认知能力（数学推理、长上下文处理、多语言能力、代码生成等）进行专项强化，数据不局限于单一领域，而是精心构建的"能力激活"数据集。

目标三：知识更新（Knowledge Update）

通过注入时效性强的新数据（近期新闻、最新研究、产品更新等），延伸模型的知识截止时间，解决"知识陈旧"问题，无需从头重新训练。

对于绝大多数企业和研究团队而言，全量重新预训练在经济上不可行。中训练提供了一种"以较小代价大幅扩展模型能力边界"的路径——使用一个开源的强基座模型（如 LLaMA-3、Mistral），针对特定需求进行中训练，往往可以在 1-2 个数量级更低的成本下获得领域专家级的性能。

6.4 代表性工作：CodeLlama、MedPaLM、DeepSeek-Coder

💻
CodeLlama（Meta, 2023）

在 LLaMA-2 基础上，使用约 500B Token 的代码数据进行中训练，其中 85% 为代码、15% 为代码相关自然语言文本。进一步细分为三个变体：CodeLlama（通用代码）、CodeLlama-Python（Python 专项）、CodeLlama-Instruct（指令跟随）。在 HumanEval 上，34B CodeLlama 超越了 GPT-3.5，展示了中训练的巨大效能。

🏥
MedPaLM / MedPaLM-2（Google, 2022/2023）

在 PaLM/PaLM-2 基础上，融合大量医学文献（PubMed、MIMIC 等）进行领域适配中训练，然后用医学问答数据进行 SFT。MedPaLM-2 在 USMLE 医师执照考试中达到"专家级"通过率（86.5%），接近人类专科医生水平，证明了中训练在高知识密度专业领域的有效性。

🔬
DeepSeek-Coder（DeepSeek, 2023）

从零预训练而非中训练，但其数据策略极具参考价值：87% 代码 + 13% 自然语言，代码涵盖 338 种编程语言。关键创新是"仓库级代码数据构建"——将 GitHub 仓库的文件按依赖关系排序后拼接，让模型学习跨文件的代码依赖关系，这一数据工程创新使其在代码补全任务上大幅领先同期竞品。

📐
Minerva（Google, 2022）

在 PaLM 基础上，使用 38.5B Token 的科学和数学数据（arXiv 论文、数学教材、科学网页）进行中训练。在 MATH 基准上达到 50.3%（当时 SOTA），在 STEM 问答上大幅超越同规模通用模型，是能力增强型中训练的经典案例。关键发现：LaTeX 格式的训练数据对数学推理能力的贡献远超纯文字描述的数学内容。

行业观察：2024 年后，中训练已成为几乎所有垂直领域 LLM 产品的标配流程。从医疗、法律、金融到教育，各类"行业大模型"的核心竞争力大多来自于精心设计的中训练数据策略，而非模型架构的创新。数据，再一次成为最关键的差异化因素。

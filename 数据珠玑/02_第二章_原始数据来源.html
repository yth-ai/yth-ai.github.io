2.1 网络爬虫数据：Common Crawl 的使用与局限

Common Crawl（CC）是全球最大的公开网络爬取数据集，由同名非营利组织自 2008 年开始维护，每月爬取约 30 亿页面，累计存档超过 250 亿页面，原始数据量超过 3PB。几乎所有已知的大型预训练语料库——GPT-3、LLaMA、Gopher、PaLM、Falcon——都以 Common Crawl 为主要数据来源。

CC 的技术结构

Common Crawl 以三种格式分发：WARC（原始 HTTP 响应）、WET（纯文本抽取结果）和 WAT（元数据）。大多数 LLM 训练使用 WET 格式作为起点，但一些高质量数据集（如 FineWeb）会从 WARC 重新提取文本，以获得更好的 HTML 解析质量。

CC 的使用流程

原始 CC 数据质量参差不齐，直接使用几乎不可能产生高质量模型。标准的 CC 处理流程包括：

  1. 语言识别：使用 fastText 或 CLD3 等工具过滤非目标语言内容
  2. URL 黑名单过滤：去除已知的成人内容、垃圾邮件网站等
  3. 质量评分：基于困惑度（Perplexity）或分类器为每个文档打分
  4. 文档级去重：MinHash LSH 等方法去除高度相似的文档
  5. 行级去重：去除大量重复的模板文本（导航栏、版权声明等）
规模参考：2024 年的 FineWeb 数据集从约 96 个 CC 快照（2013—2024）中，处理原始 WET 约 350TB，经过去重与过滤后得到约 15 万亿 Token 的最终数据集——压缩率约为 10:1。这意味着原始 CC 中约 90% 的内容在质量过滤中被丢弃。

CC 的结构性局限

⚠️ Common Crawl 的四大结构性问题

1. 英语偏差极为严重：CC 中英语内容占比约 50%，而部分低资源语言（如非洲诸语言、南亚语言）合计不足 1%，远低于其实际使用人口比例。

2. 时间偏差：CC 的爬取深度随网站流量变化，主流媒体和高访问量网站被大量重复爬取，而专业性、学术性网站的覆盖相对不足。

3. 版权与个人隐私：CC 包含大量受版权保护的内容，以及用户个人信息（电话、地址、邮箱）。这是多起 LLM 版权诉讼的核心争议点。

4. 噪声与毒性内容：网络内容天然包含仇恨言论、错误信息、色情内容等，简单的规则过滤无法完全清除，需要分类器辅助。

基于 CC 的主流处理数据集

C4（Colossal Clean Crawled Corpus）
Google，2020 · 约 750GB / 180B Token

T5 论文提出的 CC 处理版本。过滤规则简单但有效：删除包含"坏词"的行、删除非完整句子（不以标点结尾）、删除代码行等。

开源英语为主启发式过滤
RefinedWeb
Technology Innovation Institute，2023 · 约 5T Token（部分公开）

Falcon 模型的训练数据集。引入了 Trafilatura 进行高质量 HTML 文本抽取，配合严格的去重策略（URL 去重 + MinHash LSH）。论文声称仅凭 CC 数据（无书籍、代码）即可训练出接近 SOTA 的模型。

部分开源多语言高质量提取
FineWeb / FineWeb-Edu
Hugging Face，2024 · 15T Token（FineWeb）/ 1.3T Token（Edu版）

目前质量最高的公开 CC 衍生数据集。FineWeb-Edu 进一步使用 LLM 打分器筛选教育价值高的文档，在 MMLU 等知识类基准上表现极佳。完整的处理流水线已开源。

2.2 书籍与学术论文：Books3、S2ORC、ArXiv

书籍和学术论文被视为预训练数据中质量最高、信息密度最大的来源。GPT-3 原始论文明确指出，Books 数据源（Books1+Books2）对模型的长文本理解和连贯推理能力贡献显著，尽管它们在训练数据中只占约 16%。

书籍数据的价值与来源

书籍数据的独特价值在于：

  • 长文本连贯性：书籍是天然的长上下文训练材料，帮助模型学习跨段落的逻辑关系
  • 高信息密度：经过编辑出版流程的书籍，语言质量和事实准确性通常远高于网页内容
  • 多样化文体：小说、非虚构、教材、传记等多种文体，丰富了模型的语言风格库
Books3（ThePile 中的书籍子集）
EleutherAI，2021 · 约 100B Token

来自 Bibliotik 私人电子书库的 19.6 万本书籍，是 GPT-NeoX 等开源模型的重要训练数据。因版权问题于 2023 年从公开访问中撤除（Hachette Book Group 等出版商诉讼 Meta 案件中的核心数据集之一）。

版权争议高质量英语
Pile of Law / FreeLaw
来自 Harvard Caselaw Access Project 等 · 数百 GB

包含法律判决、法规文本等，是法律领域 LLM 的重要数据来源。语言正式、逻辑严密，有助于提升模型的条件推理能力。

开源法律领域

学术论文数据

S2ORC（Semantic Scholar Open Research Corpus）
Allen Institute for AI · 约 810 万篇论文全文

覆盖多学科的学术论文全文数据集，包含结构化元数据（引用关系、摘要、章节划分）。其 LaTeX 格式版本（S2ORC-LaTeX）保留了公式和表格的原始结构，对数学/科学能力训练尤为有价值。

开源多学科结构化
ArXiv（数学/物理/CS 预印本）
Cornell University · 超过 220 万篇论文（截至 2024）

ArXiv 是数学、物理、计算机科学领域预印本的主要存储库。其 LaTeX 源文件（通过 arXiv API 获取）对模型的数学推理、公式理解和科学写作能力训练极为重要。LLaMA、Gopher 等几乎所有前沿模型都包含 ArXiv 数据。

核心来源公开访问STEM 领域LaTeX 格式

书籍与论文数据的处理要点

学术论文的 LaTeX 格式处理是一个技术难点：原始 LaTeX 包含大量宏定义、图表引用、参考文献等，需要专门的解析工具（如 LaTeXML）进行清理，否则公式可能变成乱码，影响训练效果。此外，论文中的引用部分往往被完整保留，这对模型学习引用规范有帮助，但也引入了大量"参考文献格式"文本，需要合理处理。

2.3 代码数据：GitHub、The Stack、StarCoder

代码数据在现代 LLM 预训练中扮演着超出预期的重要角色。早期研究者将代码视为一个专项能力来源，但后续大量实验表明，代码数据对通用推理能力也有显著的正向迁移效果——即使是面向自然语言任务的模型，在预训练阶段加入代码数据后，在数学推理、结构化输出、工具调用等任务上的表现均有提升。

为什么代码能增强推理能力

代码具有几个独特的语言学特性，使其成为推理能力训练的天然素材：

  • 逻辑严密性：代码不允许歧义，每一行都有明确的语义，强迫模型学习精确的条件推理
  • 过程可见性：函数调用、循环、递归等结构显式展示了"分步解决问题"的过程——这与 Chain-of-Thought 推理高度吻合
  • 注释-代码对：代码注释是天然的"意图-实现"对齐数据，帮助模型理解指令跟随
GitHub（原始爬取）
通过 GitHub Archive / GH Torrent / BigQuery 获取 · 规模数 TB

大多数代码数据集的原始来源。原始 GitHub 数据需要处理许可证过滤（仅保留允许商业使用的开源许可证）、去除自动生成代码、去除无效仓库（star 数过少、README 缺失等）。

许可证复杂多语言代码
The Stack（v1 / v2）
BigCode Project，2022/2024 · v1: 6.4TB；v2: 约 900B Token（去重后）

目前最大的公开代码数据集，覆盖 300+ 编程语言。The Stack v2 改进了许可证过滤流程，支持"退出机制"（开发者可申请移除自己的代码），并引入了基于 GitHub star 的质量过滤。

2.4 多语言数据的构建策略

绝大多数互联网数据以英语为主。然而，真正有价值的 LLM 应当具备多语言能力——不仅为了服务全球用户，也因为多语言训练被证明可以提升跨语言推理和翻译能力。

多语言数据的核心挑战

长尾语言的数据稀缺：全球有约 7000 种活语言，但在互联网上有显著存在的语言不超过 100 种。许多南亚、非洲、东南亚语言的数字化文本极为有限，导致模型在这些语言上的能力严重不足。

质量与数量的矛盾：为了增加低资源语言数据量，一些数据集采用机器翻译策略，但翻译质量参差不齐，可能引入翻译腔和错误。

语言平衡策略

在多语言训练中，直接按数据量比例训练会导致英语的强势主导，压制低资源语言的学习。主流策略是采用上采样（Upsampling）低资源语言，常用方法是对语言 l 的采样概率使用指数化平滑：

多语言采样公式（mBERT / XLM-R 策略）

P(l) ∝ (n_l)^α，其中 α ∈ (0, 1)

α = 1 对应按原始数据量比例采样（不平衡）；α = 0.3~0.7 是常用的折中方案，在减小高资源语言主导的同时，为低资源语言提供更多学习机会。

2.5 专有数据源与版权风险

除公开数据源外，领先 LLM 公司普遍拥有或使用大量专有数据，这些数据往往是其核心竞争优势所在，也是法律风险的高度集中区。

典型专有数据类型

  • 对话与用户交互日志：OpenAI 的 ChatGPT 用户对话（经脱敏）、搜索引擎的搜索-点击数据
  • 专业数据库：医学文献数据库（PubMed 之外的临床记录）、法律案例数据库、金融新闻数据库
  • 新闻内容授权：部分公司与 AP、路透社、《纽约时报》等签订数据许可协议
  • 书籍授权：与出版商达成版权许可，使用完整书籍内容

版权法律态势（截至 2024 年）

⚠️ 重大版权诉讼概览

《纽约时报》诉 OpenAI & Microsoft（2023）：指控使用其文章训练 GPT 模型，要求赔偿和删除训练数据中的 NYT 内容。被广泛视为 AI 版权诉讼的标志性案件。

Hachette、HarperCollins 等出版商诉 Meta（2023）：指控 LLaMA 训练数据包含大量未经授权的书籍（来自 Books3/LibGen）。

Getty Images 诉 Stability AI（英国 & 美国，2023）：虽针对图像生成模型，但其版权理论（训练即复制）对文本 LLM 具有参考价值。

合理使用（Fair Use）争议核心：美国法院尚未就"训练 AI 是否构成版权侵权"作出最终裁决。部分法学学者认为训练属于合理使用（transformative use），另一部分则持相反意见。

数据合规的工程应对

面对日益严格的版权监管，工业界正在采取几种应对策略：

  1. 来源透明化：完整记录每份训练数据的来源 URL、获取时间、处理步骤，以备法律审查
  2. 机器可读版权标记遵从：尊重 robots.txt 和 TDMRep（Text and Data Mining Reservation Protocol）标记
  3. 退出机制（Opt-out）：如 The Stack 提供的开发者退出机制，创作者数据集 opt-out
  4. 转向合成数据：用模型生成的合成数据替代版权敏感的真实内容，规避法律风险
  5. 授权数据采购：与内容提供商签订数据许可协议
行业趋势：版权问题正在倒逼整个行业向"数据透明度"和"授权使用"的方向迁移。欧盟《人工智能法案》和《数字单一市场版权指令》要求 AI 公司披露训练数据概要，并建立版权作品使用的授权机制。未来，数据合规性将成为模型发布的强制门槛之一，而非可选项。

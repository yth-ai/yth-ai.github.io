Chapter 3 · Data Cleaning and Filtering
第3章 数据清洗与过滤
从去重到质量过滤，从毒性检测到偏差校正——构建高质量预训练语料库的核心工程实践。

3.1 去重策略：精确去重与模糊去重（MinHash LSH）

数据重复是预训练语料库中最普遍、危害最深的问题之一。Lee et al.（2022）的研究表明，未去重的 Common Crawl 数据中，相同或高度相似的文档可能以数十至数百次的频率重复出现，这会导致模型过拟合这些重复片段，并降低泛化能力。

"如果你的训练数据中，'巴黎是法国的首都'这句话出现了 50000 次，而'布鲁塞尔是比利时的首都'只出现了 500 次，模型对这两个事实的'确信程度'是截然不同的。" — 数据重复的直觉解释

精确去重（Exact Deduplication）

精确去重通过计算文档或片段的哈希值来识别完全相同的内容。常用方法：

  • 文档级哈希：对整个文档内容计算 MD5 或 SHA-256，相同哈希值即为重复文档
  • URL 去重：对于网页数据，相同 URL 的不同爬取版本通常保留最新的一份
  • 段落级哈希：对固定大小的文本窗口计算哈希，识别文档内部的重复段落（如导航栏、版权声明等模板化内容）

精确去重的局限在于：只能处理完全相同的内容，无法识别近似重复（如同一篇文章在不同网站上的轻微改编版本）。

模糊去重：MinHash LSH

MinHash LSH（Locality Sensitive Hashing）是目前最主流的大规模近似去重算法，用于识别语义上高度相似但不完全相同的文档对。

MinHash 核心原理：
1. 将文档分解为 n-gram 集合（通常使用 5-gram）
2. 对 n-gram 集合应用 k 个随机哈希函数，取每个函数的最小哈希值，构成长度为 k 的 MinHash 签名
3. Jaccard 相似度 J(A,B) ≈ P(minhash(A) == minhash(B))
4. 使用 LSH 将签名分组为 band，相同 band 的文档对即为候选重复对
5. 验证候选对的实际 Jaccard 相似度，超过阈值（如 0.8）则标记为重复

复杂度：O(n) 而非 O(n²)，使得 100T+ Token 规模的去重成为可能

去重粒度的权衡

去重粒度优势劣势典型使用场景
文档级计算效率高，不破坏文档完整性无法处理段落级重复初步去重
段落/句子级去除模板文本更彻底可能破坏文档上下文连贯性精细化清洗
n-gram 级最细粒度，去除几乎所有重复计算开销大，可能过度删除高质量精选数据集
研究发现（Lee et al., 2022）：对训练数据进行去重后，在相同 Token 数下，模型的困惑度（Perplexity）下降明显，且在下游任务的 few-shot 性能上提升显著——去重效果等效于增加约 15-20% 的数据量。

3.2 质量过滤：启发式规则 vs 分类器过滤

质量过滤是预训练数据工程中决策空间最大、对模型性能影响最直接的步骤之一。根据实现复杂度，可分为两大类：启发式规则过滤和基于机器学习的分类器过滤。

启发式规则过滤

启发式规则基于文档的统计特征，无需训练模型，计算成本低，可解释性强，但对边界案例处理能力有限。

规则类别具体规则示例针对的问题
文档长度字词数 < 50 或 > 100000过短碎片 / 异常长文
字母比例字母字符比例 < 0.7乱码 / 链接堆积
停用词覆盖常见停用词比例过低非自然语言文本
行长度平均行长度 > 300 字符（含 Minified JS 等）机器生成代码混入
重复行比例重复行占比 > 0.3模板化低质量页面
特殊字符标点比例异常高 / HTML 标签残留不完整的文本提取
数字比例数字比例 > 0.15数据表格 / 代码混入

C4（Google，2020）是使用纯启发式过滤的典型案例，规则包括：仅保留以标点符号结尾的行、删除包含特定敏感词汇的行、删除 JavaScript 包含行、删除"lorem ipsum"等占位符等。虽然规则简单，但 C4 的质量在业界仍被认可，说明合理的启发式规则有其价值。

基于分类器的质量过滤

分类器过滤通过训练一个二分类模型来区分"高质量"与"低质量"文档，能够捕捉启发式规则难以处理的复杂质量信号。

基于困惑度的过滤（Perplexity Filtering）

使用一个小型、已知高质量语料训练的语言模型（如 KenLM 训练在维基百科上），对候选文档计算困惑度。困惑度过高表明文档与高质量语言分布偏离过大，应被过滤。这一方法被 CCNet（Grave et al., 2020）广泛使用。

困惑度过滤阈值设定（CCNet 方法）：
将文档按困惑度排序，保留困惑度在 10th~50th 百分位（即中等困惑度）的文档
过低：可能是模板化重复内容（如词汇表）
过高：可能是乱码或非目标语言内容

基于 fastText 的质量分类器

训练一个 fastText 分类器，正样本为"高质量参考文本"（如维基百科、精选书籍），负样本为"低质量随机网页"，对 CC 数据中的每个文档打分。Gopher、LLaMA 等模型均使用了类似的质量分类器。

基于 LLM 的质量评分（新趋势）

FineWeb-Edu 使用 Llama-3-70B 对文档的"教育价值"进行 0~5 分打分，然后训练一个更小的分类器（基于 Llama-3.2-1B）来模拟 LLM 的打分行为，以可接受的成本对整个 FineWeb（15T Token）进行评分。这代表了"LLM-as-quality-judge"的最新趋势。

关键对比（来自 DCLM，2024）：在相同数据规模下，使用基于 fastText 的质量分类器过滤的数据集，训练出的模型在 MMLU（5-shot）上得分比纯启发式过滤提升约 4-7 个百分点。这证明分类器过滤在质量上明显优于启发式方法，代价是需要预先准备高质量正样本。

3.3 毒性内容过滤与安全考量

互联网数据中包含大量仇恨言论、歧视内容、色情内容、暴力内容和个人隐私信息。这些内容如果进入预训练语料，会使模型学习到有害的语言模式，在后续使用中产生安全风险。

毒性内容的主要类型

  • 仇恨言论（Hate Speech）：针对种族、宗教、性别、性取向等的歧视性表达
  • 有害信息（Harmful Information）：武器制造、自杀方法、毒品制作等
  • 色情内容（Adult Content）：明确性描述，特别是涉及未成年人的内容
  • 个人隐私（PII）：姓名+电话/地址组合、身份证号码、医疗记录等
  • 错误信息（Misinformation）：已被证伪的谣言、阴谋论等

毒性过滤的工程方法

词汇黑名单过滤：维护一个包含几千到几万条有害词汇/短语的黑名单，过滤包含这些内容的文档。这是最简单、速度最快的方法，但误报率（将正常内容过滤掉）和漏报率都较高。

专用毒性分类器：Perspective API（Google）、Detoxify、HateBERT 等专门训练的毒性检测模型，能识别更微妙的有害表达，但计算成本高，不适合直接用于 TB 级数据的全量过滤，通常用于抽样验证。

URL 域名黑名单：维护已知的成人内容网站、极端主义论坛等的域名黑名单，在 URL 级别过滤，是效率最高的大规模毒性内容过滤手段之一。LDNOOBW（List of Dirty, Naughty, Obscene, and Otherwise Bad Words）等公开词表被广泛使用。

安全与多样性的张力

⚠️ 过度过滤的风险

毒性过滤存在显著的过度过滤风险：基于词汇黑名单的方法会将"癌症"、"枪支历史"、"性别研究"等合法内容误删；基于主题的过滤可能不成比例地删除边缘群体（LGBTQ+）的内容，加剧代表性偏差。Welbl et al.（2021）的研究表明，激进的毒性过滤会使模型对少数群体话题的了解能力显著下降。

目前业界的主流实践是：对明显有害内容（儿童性剥削、武器制造指南等）采取零容忍过滤，而对灰色地带内容（如涉及暴力的文学作品、批评性讨论歧视现象的文章等）采取更保守的过滤策略，并在后训练阶段通过 RLHF 和安全微调来处理行为对齐问题。

3.4 数据集代表性偏差的识别与校正

即使完成了去重和毒性过滤，预训练数据仍可能携带严重的代表性偏差——某些群体、观点、文化或语言在数据中被过度或不足表示，导致模型学习到不公平的关联或刻板印象。

主要偏差类型

性别偏差：互联网文本中，"护士"与"女性"、"工程师"与"男性"等强关联大量存在，模型可能习得这些刻板印象。Bolukbasi et al.（2016）的开创性工作揭示了词向量中的性别偏差，这一问题在 LLM 时代更为复杂。

地理与文化偏差：互联网内容高度集中于英语国家和发达地区，模型对这些地区的文化、历史和价值观的理解远深于其他地区。

时间偏差：历史上，互联网普及之前的知识（前 2000 年代）在数字文本中的表示较少，导致模型对近代史和当代事件的了解远超古代历史。

观点与立场偏差：如果训练数据中特定政治立场的内容显著多于其他立场，模型会表现出偏向。

偏差的识别方法

  • 词共现分析：分析敏感属性词（性别、种族词汇）与职业/特征词的共现频率
  • 模板化测试集：使用 WinoBias、StereoSet 等专门设计的偏差评测集评估模型行为
  • 话题分布分析：对训练数据进行主题建模，量化不同话题的覆盖比例
  • 地理与人口统计分析：分析文本所提及的地名、人名的地理分布

校正策略的局限性

现实认知：完全消除预训练数据中的偏差在技术上是不可能的——任何数据集都是对真实世界的采样，而真实世界本身就存在不平等和不均匀分布。预训练阶段的"偏差校正"更多是缓解最严重的偏差，真正的行为对齐需要在后训练（RLHF、安全微调）阶段进行。这也是预训练数据工程与对齐研究之间的分工边界。

3.5 清洗流水线工程实践：Dolma、RedPajama、FineWeb 案例

将上述各类过滤技术组合为高效、可扩展的清洗流水线，是预训练数据工程的核心挑战。以下三个代表性开源项目提供了完整的工程参考。

案例一

Dolma（Allen Institute for AI，2024）

规模：约 3 万亿 Token，用于训练 OLMo 系列模型

数据源构成：Common Crawl（67%）、The Stack（11%）、C4（10%）、Reddit（5%）、维基百科/Wikibooks（4%）、arXiv（3%）

核心清洗步骤：

1
URL 过滤：UTI（Uniform Tags for Identification）格式的 URL 黑名单，覆盖成人、垃圾、钓鱼等分类
2
语言过滤：fastText 模型，保留英语为主
3
质量过滤：基于 Gopher 规则的启发式过滤 + fastText 质量分类器双重过滤
4
毒性过滤：Jigsaw Perspective API 采样验证 + 词汇黑名单全量过滤
5
PII 脱敏：正则表达式识别并替换电话号码、邮件地址、IP 地址等
6
模糊去重：Bloom Filter + MinHash LSH 两阶段去重

工程亮点：所有工具以 Python 包形式开源发布，支持分布式处理；提供详细的数据审查报告（每个数据子集的统计信息、过滤率）。

案例二

RedPajama-v2（Together AI，2023）

规模：约 30 万亿原始 Token（最大规模的公开 CC 处理数据集），附带质量信号注释

创新设计：RedPajama-v2 的独特之处在于，它提供了原始文档 + 多维质量信号（约 40 项）的分离存储，允许下游用户根据自己的需要自定义过滤阈值，而不必重新处理原始数据。质量信号包括：

  • 困惑度分（基于 CCNet KenLM 模型）
  • 自然语言比例（Natural Language Score）
  • 多项启发式质量指标（行重复比例、特殊字符比例等）
  • DSIR（Data Selection via Importance Resampling）相似度分

工程价值：将"数据存储"与"过滤策略"解耦，是大规模数据集分发的重要创新，允许研究社区在同一个数据集上进行不同过滤策略的对比实验。

案例三

FineWeb（Hugging Face，2024）

规模：15 万亿 Token（FineWeb）+ 1.3 万亿 Token（FineWeb-Edu）

核心清洗流水线（datatrove 框架）：

1
高质量 HTML 解析：使用 trafilatura 重新解析 WARC 文件（而非直接使用 WET），显著提升文本抽取质量
2
URL 黑名单 + 语言过滤：fastText 语言识别，保留英语（FineWeb）或多语言（FineWeb-2）
3
Gopher 质量规则：约 20 项启发式规则，过滤明显低质量文档
4
C4 质量规则：补充 C4 的过滤规则集
5
MinHash 文档级去重：5-gram MinHash，Jaccard 相似度阈值 0.7
6
（FineWeb-Edu 附加）LLM 教育质量评分：Llama-3-70B 打分，训练 1B 分类器模拟推断

Ablation 研究：FineWeb 发布了完整的消融实验，证明每个过滤步骤对最终模型性能的贡献，是业界罕见的透明度示范。

流水线设计的通用原则

工程最佳实践总结：
  1. 先粗后细：先用低成本的 URL 黑名单和启发式规则大量过滤，再用高成本的分类器精细处理
  2. 记录保存过滤日志：每步过滤保留被删除文档的统计信息，便于后续分析和调整
  3. 去重放在最后：在质量过滤完成后再去重，避免将低质量版本保留而删除高质量版本
  4. 消融实验验证：每个过滤步骤应通过小规模训练实验验证其对模型性能的实际贡献
  5. 分布式处理架构：使用 Apache Spark、Dask 或 datatrove 等框架，确保处理流水线可横向扩展至 PB 级数据

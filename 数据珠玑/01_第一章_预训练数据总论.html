Chapter 1 · Pre-training Data Overview
第1章 预训练数据总论
从数据的本质定义出发，理解规模演进、最优配比法则，以及贯穿整个预训练阶段的核心权衡。

1.1 预训练数据的本质：语言分布的近似

理解预训练数据，需要先理解语言模型在做什么。从数学角度，语言模型的核心目标是学习一个概率分布 P(w₁, w₂, ..., wₙ)——即自然语言的联合概率分布。预训练，就是在海量文本上最大化这个分布的似然函数。

语言建模目标（自回归）：
max Σ log P(wₜ | w₁, w₂, ..., wₜ₋₁; θ)

直觉解读：模型参数 θ 本质上是在"压缩"训练数据所代表的语言分布。

这个看似简单的目标，蕴含了一个深刻的哲学命题：训练数据的分布，就是模型所学习的"世界的样子"。模型不会比它见过的文本更聪明，也无法超越训练数据的知识边界——它只能在训练数据所代表的语言空间中进行插值和有限的外推。

语言分布的三个维度

当我们说"训练数据是语言分布的近似"时，这个近似至少包含三个维度：

1. 主题分布（Topic Distribution）：训练数据覆盖了哪些主题领域？比例如何？如果网络爬取数据中，科技话题占 40%，娱乐占 20%，医疗占 5%，那么模型在这些领域的知识深度大体上也会呈现相似的比例关系。

2. 文体分布（Style Distribution）：包括正式文本与口语、长文与短文、结构化与非结构化、编辑性文本（书籍、新闻）与用户生成内容（社交媒体、论坛）等维度。文体分布直接影响模型的写作风格、指令遵从能力和格式化输出能力。

3. 时间分布（Temporal Distribution）：训练数据的时间跨度决定了模型的知识截止时间，也影响模型对不同历史时期事件的了解程度。

核心命题：预训练数据工程的本质，是对人类知识与语言的有意识采样。每一个数据决策——选哪些源、排除哪些内容、各自占多少比例——都是在塑造模型将要内化的"世界模型"。这不是纯粹的技术问题，也是认识论和价值观的问题。

涌现能力与数据的关系

2022 年 Wei et al. 提出的"涌现能力"（Emergent Abilities）现象表明，某些能力在模型规模超过某个阈值后会突然出现。后续研究逐渐揭示：涌现能力不仅取决于模型规模，也取决于训练数据中是否包含足够多的相关任务示例。例如，代码生成能力的涌现，与训练数据中代码比例高度相关；多步推理能力的出现，与数学和逻辑文本的充分覆盖密不可分。

1.2 数据规模的演进：从 GB 到 Trillion Token

过去五年，预训练数据规模经历了指数级增长。这一增长不是线性的，而是伴随着理论认知的进步和工程能力的突破，呈现出几个清晰的跃迁节点。

2018
GPT-1：BooksCorpus，约 5GB、8 亿 Token。奠定了预训练+微调的范式，但数据规模极其有限。
2019
GPT-2：WebText，约 40GB、100 亿 Token。引入网络爬取数据，首次展示了数据规模对生成质量的显著提升。
2020
GPT-3：~570GB 过滤后文本，约 3000 亿 Token。首次将书籍、维基百科、网页爬取数据混合，并公开了数据配比（WebText2 22%、Books1 8%、Books2 8%、Wikipedia 3%、CommonCrawl 60%）。
2022
PaLM / Chinchilla：Chinchilla 用 1.4 万亿 Token 训练 700 亿参数模型；PaLM 使用约 7800 亿 Token。数据规模开始以 Trillion 为单位描述。
2023
LLaMA-1/2：LLaMA-1 使用约 1.4 万亿 Token；LLaMA-2 约 2 万亿 Token。开源社区开始主动构建 RedPajama、Dolma 等对应的开放数据集，推动了透明度。
2024
LLaMA-3 / Mistral / DeepSeek：LLaMA-3 使用约 15 万亿 Token，FineWeb 等公开数据集跨越 15T Token。DeepSeek-V3 训练数据达 14.8 万亿 Token，并引入质量分类和复杂推理数据增强。

1.3 数据与算力的最优配比：Chinchilla 法则及其争议

2022 年，DeepMind 的 Hoffman 等人发表了《Training Compute-Optimal Large Language Models》（又称"Chinchilla 论文"），在 LLM 领域引发了一场范式地震。

Chinchilla 法则的核心结论

在固定计算预算（FLOPs）下，模型参数量（N）和训练 Token 数（D）应该等比例扩展——每个参数对应约 20 个 Token 的训练量，才是计算效率最优的配置。

Chinchilla 最优配比（近似）：
N_opt ∝ C^0.5
D_opt ∝ C^0.5

其中 C 是总计算量（FLOPs）。约束条件：D ≈ 20 × N

对比 Kaplan 等人（2020）的结论：D ∝ C^0.27（大幅低估了数据需求）

这个结论的冲击力在于：它说明 GPT-3（1750 亿参数，3000 亿 Token）严重欠训练（undertrained）——按 Chinchilla 配比，同等算力应当训练一个约 70 亿参数的模型，使用 1.4 万亿 Token。DeepMind 随即验证：70B 的 Chinchilla 模型在大多数基准上超越了 280B 的 Gopher。

法则的局限与争议

⚠️ Chinchilla 法则的五个重要局限

1. 以训练损失为优化目标，非推理效率：Chinchilla 的"最优"是在固定算力下最小化验证集损失，但工业界的实际目标通常是"固定推理成本下最大化能力"。这一差异导致 LLaMA 系列有意选择比 Chinchilla 更小的模型、更多的数据。

2. 数据质量假设过于理想：Chinchilla 实验使用的 MassiveText 数据集质量相对均匀。现实中，数据质量差异巨大，低质量 Token 不等于高质量 Token。

3. 未考虑数据重复的影响：Muennighoff et al.（2023）发现，当优质数据不足时，对数据进行多次 Epoch 重复训练，可以部分弥补数据量的不足，且损失可控。

4. 仅适用于标准自回归预训练：对于 MoE 架构、多模态模型，Chinchilla 法则的适用性存疑。

5. 幂律指数可能随数据分布变化：不同数据混合比例下，Scaling 曲线的斜率可能不同，导致最优配比发生漂移。

2024 年，随着 LLaMA-3 等"过训练"模型的成功，业界对 Chinchilla 法则的理解逐渐成熟：它描述的是训练计算效率的最优点，而非模型能力的绝对上限。在数据充足的前提下，超出 Chinchilla 最优点继续训练，仍能提升模型能力，只是边际算力回报递减。

1.4 数据质量 vs 数据数量的权衡

如果说 Chinchilla 法则解决了"数量"的问题，那么数据质量则是预训练数据工程中最难量化、争议最多的维度。

质量的多义性

"数据质量"在不同语境下指代不同的事物：

信息密度（Informativeness）：每 Token 包含多少有效信息？一篇经过编辑的百科全书文章，信息密度通常远高于重复率高的网页广告文本。

语言规范性（Linguistic Quality）：是否存在大量语法错误、乱码、机器翻译痕迹？

事实准确性（Factual Accuracy）：训练文本中的事实性内容是否可靠？这对模型的"幻觉"（Hallucination）率有直接影响。

多样性（Diversity）：训练数据是否覆盖了足够多样化的主题、风格、观点？高质量不等于同质化——单一的"高质量"数据来源可能造成能力偏窄。

几个典型的质量-数量权衡案例

案例一：phi-1 与"教科书级数据"（Textbooks Are All You Need）

微软 2023 年发布的 phi-1 仅用约 6.7B Token 的精选合成数据（模拟教科书质量）和 Python 代码，训练出了在 HumanEval 上超越大多数依赖数百亿 Token 的代码模型的性能。这一工作最直接地证明了：高质量数据可以在很大程度上抵消数量劣势。

案例二：FineWeb 与大规模质量过滤

Hugging Face 2024 年发布的 FineWeb 数据集（15 万亿 Token）通过多轮质量过滤（去重 + 基于 fastText 的质量分类器），在保持超大规模的同时，使 MMLU、HellaSwag 等基准上的模型性能明显优于未过滤的同量数据。FineWeb 的实验表明：质量过滤和规模扩展并非二选一，可以协同实现。

案例三：DCLM 的跨数据集比较

DataComp for Language Models（DCLM，2024）是一个系统性的数据集对比研究，在控制算力的前提下，比较了不同过滤策略的数据集所训练出的模型性能。结论显示：使用基于分类器的质量过滤（而非纯粹的启发式规则）可以带来显著的下游性能提升，且效果稳定跨越多个评测集。

质量与数量的非线性关系

研究表明，数据质量对模型性能的影响呈现明显的非线性特征：

  • 极低质量的数据（垃圾文本、重复内容、无意义序列）对训练有负向影响，"减去"这些数据比"加入"等量高质量数据带来的增益更大。
  • 在"基本可用"的数据范围内，数量与质量的互补性较强——可以用更多数据弥补质量的不足，也可以用更高质量数据减少数量需求。
  • 但在追求极端性能（state-of-the-art）时，质量的作用呈现超线性增益——顶尖模型通常都会对数据质量进行极为细致的控制。
实践原则：对于资源有限的团队，数据质量优先于数据数量。清洗 100B 高质量 Token 训练的模型，通常优于用 1T 低质量 Token 训练的同规模模型。但对于资源充足、追求前沿性能的团队，两者都需要极致——质量过滤 + 规模扩展是目前最有效的组合路径。

章节小结

本章建立了理解预训练数据的基础框架：数据是语言分布的近似，其规模从 GB 演进到 Trillion Token；Chinchilla 法则提供了算力-数据配比的理论指导，但工业实践已超越这一框架；数据质量与数量之间存在复杂的权衡关系，最优策略因资源约束和目标而异。后续章节将在这一框架基础上，深入探讨数据来源、清洗、配比和前沿问题。

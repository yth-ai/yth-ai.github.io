Agent能力的数据优化路径

本报告系统梳理大模型Agent能力优化路径，聚焦预训练/中训练数据角度，分析Agent类型、数据类型、优化策略，整合业界实践和学术前沿技术。

核心定义

中训练（Mid-training）：在通用预训练完成后、任务后训练开始前，以中等规模的计算预算，在精心设计的数据分布上继续优化模型，以系统性增强特定能力，同时尽量维持甚至改善原有通用能力的过程。

1. Agent类型分类矩阵

从应用领域和数据需求角度，将Agent分为四大类型，每种类型有独特的能力聚焦和数据需求特征。

通用Agent

多任务泛化工具使用长时记忆

能力聚焦：跨领域任务执行、多工具协调、长期上下文理解

数据需求：多样化任务轨迹、多工具环境数据、长对话历史

Code Agent

代码理解环境交互调试恢复

能力聚焦：代码理解与生成、执行环境交互、错误调试与恢复

数据需求：Github数据、测试环境、执行反馈、错误恢复轨迹

Research Agent

信息检索分析综合假设生成

能力聚焦：信息检索与评估、多源信息综合、假设生成与验证

数据需求：科学文献、数据图表、研究流程、评估标准

领域Agent

领域语法专业逻辑约束遵循

能力聚焦：领域专有语法、专业逻辑推理、规则约束遵循

数据需求：行业文档、领域知识库、专业论坛、专家决策轨迹

2. 数据类型体系

根据训练阶段划分数据架构，不同Agent类型在不同训练阶段有差异化数据需求。

训练阶段 数据性质 核心数据内容 关键挑战
预训练 通用知识 自然语料、代码语料、多模态、知识图谱 数据质量、规模、多样性平衡
中训练 能力专精 轨迹合成、工具使用、推理链、长上下文、多Agent交互 数据真实性、可验证性、难度控制
后训练 任务对齐 特定任务轨迹、人工标注、强化学习奖励 奖励稀疏性、人工成本、泛化能力

中训练数据的关键类型

轨迹合成数据：模拟真实交互轨迹，包含错误恢复和策略选择

工具使用数据：API调用、环境交互、工具链组合

推理链（CoT）数据：思维链、决策过程、验证步骤

长上下文数据：长文档理解、多轮对话、记忆保持

多Agent交互数据：协作模式、通信协议、分布式决策

3. 数据优化策略

针对不同Agent类型的数据需求，采用差异化优化策略提升能力和泛化性。

数据合成

基于规则LLM引导环境驱动

通过规则系统、LLM生成器和环境模拟器合成多样化训练数据

质量控制

多样性难度控制可验证性

确保数据覆盖不同难度级别、场景变体和可验证的解决方案

课程学习

渐进式退火策略能力分层

从简单到复杂的训练顺序，逐步引入新能力和复杂场景

Code Agent优化要点

1. 高保真环境合成：TermiGen通过迭代多Agent精炼循环创建可执行Docker环境

2. 鲁棒轨迹收集：Generator-Critic协议主动注入错误，训练错误恢复能力

3. 数据多样性：覆盖不同编程语言、框架、工具链和项目规模

4. 业界实践案例

各大公司在中训练阶段实施的数据优化策略各有侧重，形成不同的技术路线。

公司/模型 技术特色 数据配方 效果验证

关键洞察：中训练阶段加入agentic数据可以让模型先学会各种工具使用，在RL阶段只需专注具体任务，提高采样效率。相同数据下，在CPT阶段训练比SFT更能学会难题。

5. 学术前沿技术

2025-2026年出现的创新数据合成方法和技术框架。

ScaleEnv（2602.06820）

核心创新：从零开始合成环境用于通用工具使用Agent训练

技术特点：程序测试确保可靠性 + 工具依赖图扩展保证任务完整性

效果验证：在τ²-Bench和VitaBench上显著提升泛化能力

AgentSkiller（2602.09372）

核心创新：语义集成跨领域数据合成提升通用Agent智能

技术特点：DAG架构 + Person-Centric Entity Graph + Service Blueprints

效果验证：合成~11K交互样本，显著提升函数调用能力

RA3算法

核心创新：通过action abstraction压缩决策空间

技术特点：在代码推理轨迹上引入latent thought variables

效果验证：通过训练抽象动作，提升下游RL收敛速度和性能
